<section id="overview-and-architecture" class="level2">
<h2 class="anchored" data-anchor-id="overview-and-architecture" id="overview-and-architecture">Overview and Architecture</h2>
<p>This guide walks through the full pipeline for identifying images your model predicts incorrectly, storing and querying them using <strong>Milvus</strong> as the vector database, re-annotating them with correct labels, extending your training dataset, and fine-tuning the model so it learns from its own mistakes.</p>
<p>Milvus plays a central role throughout: it stores dense float embeddings from your model’s penultimate layer alongside scalar metadata (ground truth label, predicted label, confidence, annotation status), and lets you query them with both <strong>scalar filters</strong> and <strong>vector search</strong> for finding the most diverse and informative failure cases.</p>
<section id="why-milvus-for-this-pipeline" class="level3">
<h3 class="anchored" data-anchor-id="why-milvus-for-this-pipeline" id="why-milvus-for-this-pipeline">Why Milvus for This Pipeline?</h3>
<p>Traditional approaches store predictions in flat CSV files and run pandas queries to find errors. This breaks down when your dataset grows beyond a few hundred thousand images, when you want to find visually similar hard examples rather than just statistically similar ones, when you need to deduplicate hard examples to avoid adding hundreds of near-identical failure images, and when you are running multiple fine-tuning iterations and need to track annotation state per sample across iterations.</p>
<p>Milvus solves all of these. Scalar filters handle rule-based selection and ANN search handles embedding-space diversity sampling, both against the same store.</p>
</section>

<section id="greedy-furthest-point-sampling" class="level3">
<h3 class="anchored" data-anchor-id="greedy-furthest-point-sampling" id="greedy-furthest-point-sampling">Greedy Furthest-Point Sampling</h3>
<div id="09e04f6d" class="cell" data-execution_count="10">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb12"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb12-1"><a href="#cb12-1" aria-hidden="true" tabindex="-1"></a><span class="kw">def</span> greedy_diverse_select(candidates, embeddings_map: <span class="bu">dict</span>, budget: <span class="bu">int</span>):</span>
<span id="cb12-2"><a href="#cb12-2" aria-hidden="true" tabindex="-1"></a>    selected      <span class="op">=</span> [candidates[<span class="dv">0</span>]]</span>
<span id="cb12-3"><a href="#cb12-3" aria-hidden="true" tabindex="-1"></a>    selected_embs <span class="op">=</span> [embeddings_map[candidates[<span class="dv">0</span>].<span class="bu">id</span>]]</span>
<span id="cb12-4"><a href="#cb12-4" aria-hidden="true" tabindex="-1"></a>    remaining     <span class="op">=</span> candidates[<span class="dv">1</span>:]</span>
<span id="cb12-5"><a href="#cb12-5" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb12-6"><a href="#cb12-6" aria-hidden="true" tabindex="-1"></a>    <span class="cf">while</span> <span class="bu">len</span>(selected) <span class="op">&lt;</span> budget <span class="kw">and</span> remaining:</span>
<span id="cb12-7"><a href="#cb12-7" aria-hidden="true" tabindex="-1"></a>        best_idx  <span class="op">=</span> <span class="op">-</span><span class="dv">1</span></span>
<span id="cb12-8"><a href="#cb12-8" aria-hidden="true" tabindex="-1"></a>        best_dist <span class="op">=</span> <span class="op">-</span><span class="fl">1.0</span></span>
<span id="cb12-9"><a href="#cb12-9" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb12-10"><a href="#cb12-10" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> i, cand <span class="kw">in</span> <span class="bu">enumerate</span>(remaining):</span>
<span id="cb12-11"><a href="#cb12-11" aria-hidden="true" tabindex="-1"></a>            emb      <span class="op">=</span> embeddings_map[cand.<span class="bu">id</span>]</span>
<span id="cb12-12"><a href="#cb12-12" aria-hidden="true" tabindex="-1"></a>            min_dist <span class="op">=</span> <span class="bu">min</span>(</span>
<span id="cb12-13"><a href="#cb12-13" aria-hidden="true" tabindex="-1"></a>                np.linalg.norm(emb <span class="op">-</span> s) <span class="cf">for</span> s <span class="kw">in</span> selected_embs</span>
<span id="cb12-14"><a href="#cb12-14" aria-hidden="true" tabindex="-1"></a>            )</span>
<span id="cb12-15"><a href="#cb12-15" aria-hidden="true" tabindex="-1"></a>            <span class="cf">if</span> min_dist <span class="op">&gt;</span> best_dist:</span>
<span id="cb12-16"><a href="#cb12-16" aria-hidden="true" tabindex="-1"></a>                best_dist <span class="op">=</span> min_dist</span>
<span id="cb12-17"><a href="#cb12-17" aria-hidden="true" tabindex="-1"></a>                best_idx  <span class="op">=</span> i</span>
<span id="cb12-18"><a href="#cb12-18" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb12-19"><a href="#cb12-19" aria-hidden="true" tabindex="-1"></a>        chosen <span class="op">=</span> remaining.pop(best_idx)</span>
<span id="cb12-20"><a href="#cb12-20" aria-hidden="true" tabindex="-1"></a>        selected.append(chosen)</span>
<span id="cb12-21"><a href="#cb12-21" aria-hidden="true" tabindex="-1"></a>        selected_embs.append(embeddings_map[chosen.<span class="bu">id</span>])</span>
<span id="cb12-22"><a href="#cb12-22" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb12-23"><a href="#cb12-23" aria-hidden="true" tabindex="-1"></a>    <span class="cf">return</span> selected</span>
<span id="cb12-24"><a href="#cb12-24" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb12-25"><a href="#cb12-25" aria-hidden="true" tabindex="-1"></a>emb_map        <span class="op">=</span> {r[<span class="st">"id"</span>]: np.array(r[<span class="st">"embedding"</span>]) <span class="cf">for</span> r <span class="kw">in</span> errors_with_emb}</span>
<span id="cb12-26"><a href="#cb12-26" aria-hidden="true" tabindex="-1"></a>diverse_subset <span class="op">=</span> greedy_diverse_select(candidates, emb_map, budget<span class="op">=</span>BUDGET)</span>
<span id="cb12-27"><a href="#cb12-27" aria-hidden="true" tabindex="-1"></a><span class="bu">print</span>(<span class="ss">f"Diverse hard examples selected: </span><span class="sc">{</span><span class="bu">len</span>(diverse_subset)<span class="sc">}</span><span class="ss">"</span>)</span></code></pre></div></div>

<span id="cb23-47"><a href="#cb23-47" aria-hidden="true" tabindex="-1"></a>        )</span>

<span id="cb23-50"><a href="#cb23-50" aria-hidden="true" tabindex="-1"></a>    <span class="cf">for</span> images, labels <span class="kw">in</span> tqdm(train_loader, desc<span class="op">=</span><span class="ss">f"E</span><span class="sc">{</span>epoch<span class="sc">:02d}</span><span class="ss"> train"</span>):</span>

<span id="cb23-54"><a href="#cb23-54" aria-hidden="true" tabindex="-1"></a>        optimizer.step()</span>

<span id="cb23-59"><a href="#cb23-59" aria-hidden="true" tabindex="-1"></a>    <span class="cf">with</span> torch.no_grad():</span>
<span id="cb23-60"><a href="#cb23-60" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> images, labels <span class="kw">in</span> val_loader:</span>
<span id="cb23-61"><a href="#cb23-61" aria-hidden="true" tabindex="-1"></a>            preds    <span class="op">=</span> model(images.to(DEVICE)).argmax(<span class="dv">1</span>).cpu()</span>

<span id="cb23-67"><a href="#cb23-67" aria-hidden="true" tabindex="-1"></a></span>

<span id="cb23-69"><a href="#cb23-69" aria-hidden="true" tabindex="-1"></a>        best_val_acc <span class="op">=</span> val_acc</span>

<h2 class="anchored" data-anchor-id="step-10-closing-the-active-learning-loop" id="step-10-closing-the-active-learning-loop">Step 10 — Closing the Active Learning Loop</h2>

<section id="avoid-catastrophic-forgetting" class="level3">
<h3 class="anchored" data-anchor-id="avoid-catastrophic-forgetting" id="avoid-catastrophic-forgetting">Avoid Catastrophic Forgetting</h3>
<p>Always mix hard examples with original training data. A safe starting ratio is one hard example for every three original training samples. Never fine-tune exclusively on the hard-example set.</p>

<section id="summary" class="level2">
<h2 class="anchored" data-anchor-id="summary" id="summary">Summary</h2>
<p>The Milvus-backed pipeline replaces flat CSV files with a queryable, searchable vector store that scales to millions of images. Inference inserts embeddings and prediction metadata into Milvus in batches. Scalar filters find wrong predictions and high-confidence errors with simple expressions. ANN search from the error-cluster centroid combined with greedy diversity selection surfaces the most informative and non-redundant hard examples. Delete plus re-insert writes human-corrected labels back to Milvus, keeping annotation state consistent across iterations. Dataset assembly pulls training records and annotated hard examples from Milvus directly into a manifest CSV which feeds a WeightedRandomSampler-backed DataLoader. Loop iteration tags let you compare model performance before and after fine-tuning directly in Milvus without any external tracking tool.</p>
