<section id="overview-and-architecture" class="level2">
<h2 class="anchored" data-anchor-id="overview-and-architecture" id="overview-and-architecture">Overview and Architecture</h2>
<p>This guide covers how to build a full machine learning pipeline that combines <strong>PyTorch</strong> for model training with <strong>Milvus</strong> as a vector database — a common architecture for embedding-based retrieval, semantic search, recommendation systems, and similarity learning.</p>
<section id="why-combine-milvus-and-pytorch" class="level3">
<h3 class="anchored" data-anchor-id="why-combine-milvus-and-pytorch" id="why-combine-milvus-and-pytorch">Why Combine Milvus and PyTorch?</h3>
<p>PyTorch is the de facto framework for training deep learning models. Milvus is a purpose-built vector database optimized for storing, indexing, and querying high-dimensional embeddings at scale. Together they enable architectures that were previously impractical:</p>
<ul>
<li><strong>Retrieval-Augmented Training (RAT)</strong>: During training, fetch the most similar embeddings from Milvus to use as hard negatives or context.</li>
<li><strong>Approximate Nearest Neighbor (ANN) Search</strong>: After training, your model’s outputs can be indexed in Milvus for millisecond-latency semantic search across billions of vectors.</li>
<li><strong>Continual Learning</strong>: As your model improves, periodically re-index updated embeddings without retraining from scratch.</li>
<li><strong>Few-Shot and Metric Learning</strong>: Use Milvus to retrieve labeled examples at inference time, enabling k-NN classification without retraining.</li>
</ul>
</section>

<section id="prerequisites" class="level2">
<h2 class="anchored" data-anchor-id="prerequisites" id="prerequisites">Prerequisites</h2>
<p>Before starting, make sure you are comfortable with the following:</p>
<ul>
<li><strong>Python 3.8+</strong> — all code examples use Python 3.10 idioms.</li>
<li><strong>Basic PyTorch</strong> — tensors, <code>nn.Module</code>, <code>DataLoader</code>, training loops.</li>
<li><strong>Docker</strong> — Milvus is most easily run via Docker Compose.</li>
<li><strong>Linear algebra fundamentals</strong> — understanding of cosine similarity and dot products helps when choosing distance metrics.</li>
</ul>

<section id="understanding-milvus" class="level2">
<h2 class="anchored" data-anchor-id="understanding-milvus" id="understanding-milvus">Understanding Milvus</h2>
<section id="core-concepts" class="level3">
<h3 class="anchored" data-anchor-id="core-concepts" id="core-concepts">Core Concepts</h3>
<p>Before writing any code it is important to understand how Milvus organizes data:</p>
<p><strong>Collection</strong> The top-level container in Milvus, analogous to a table in a relational database. A collection has a fixed schema that defines the fields stored for each entity.</p>
<p><strong>Schema</strong> Defines the structure of a collection. Every collection must have a primary key field and at least one <code>FLOAT_VECTOR</code> or <code>BINARY_VECTOR</code> field.</p>
<p><strong>Partition</strong> An optional subdivision of a collection. Partitions let you scope queries to a subset of data (e.g., partitioning by category or split — <code>train</code>, <code>val</code>, <code>test</code>).</p>
<p><strong>Index</strong> An ANN data structure built over a vector field. Without an index, Milvus falls back to brute-force search. Common index types:</p>
<table class="caption-top table">
<colgroup>
<col style="width: 33%">
<col style="width: 33%">
<col style="width: 33%">
</colgroup>
<thead>
<tr class="header">
<th>Index</th>
<th>Description</th>
<th>Best For</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><code>FLAT</code></td>
<td>Exact brute-force search</td>
<td>Small datasets (&lt;1M), maximum recall needed</td>
</tr>
<tr class="even">
<td><code>IVF_FLAT</code></td>
<td>Inverted file, flat quantization</td>
<td>General-purpose, medium datasets</td>
</tr>
<tr class="odd">
<td><code>IVF_SQ8</code></td>
<td>IVF with scalar quantization</td>
<td>Memory-constrained environments</td>
</tr>
<tr class="even">
<td><code>HNSW</code></td>
<td>Hierarchical Navigable Small World</td>
<td>High-recall, high-throughput production use</td>
</tr>
<tr class="odd">
<td><code>DISKANN</code></td>
<td>Disk-based ANN</td>
<td>Billion-scale datasets that don’t fit in RAM</td>
</tr>
</tbody>
</table>
<p><strong>Metric Type</strong> The distance function used when comparing vectors. Your choice here must match how your model was trained:</p>
<ul>
<li><code>L2</code> — Euclidean distance (lower = more similar)</li>
<li><code>IP</code> — Inner product / cosine similarity (higher = more similar, when vectors are L2-normalized)</li>
<li><code>COSINE</code> — Cosine similarity (Milvus 2.4+, vectors normalized automatically)</li>
</ul>
</section>

<span id="cb9-16"><a href="#cb9-16" aria-hidden="true" tabindex="-1"></a>        <span class="co"># Load backbone</span></span>
<span id="cb9-17"><a href="#cb9-17" aria-hidden="true" tabindex="-1"></a>        weights <span class="op">=</span> models.ResNet50_Weights.IMAGENET1K_V2 <span class="cf">if</span> pretrained <span class="cf">else</span> <span class="va">None</span></span>
<span id="cb9-18"><a href="#cb9-18" aria-hidden="true" tabindex="-1"></a>        backbone <span class="op">=</span> models.resnet50(weights<span class="op">=</span>weights)</span>
<span id="cb9-19"><a href="#cb9-19" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb9-20"><a href="#cb9-20" aria-hidden="true" tabindex="-1"></a>        <span class="co"># Remove the original classification head</span></span>
<span id="cb9-21"><a href="#cb9-21" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>.backbone <span class="op">=</span> nn.Sequential(<span class="op">*</span><span class="bu">list</span>(backbone.children())[:<span class="op">-</span><span class="dv">1</span>])</span>
<span id="cb9-22"><a href="#cb9-22" aria-hidden="true" tabindex="-1"></a>        backbone_out_dim <span class="op">=</span> <span class="dv">2048</span>  <span class="co"># ResNet-50 penultimate layer</span></span>
<span id="cb9-23"><a href="#cb9-23" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb9-24"><a href="#cb9-24" aria-hidden="true" tabindex="-1"></a>        <span class="co"># Projection head: 2048 -&gt; embed_dim</span></span>
<span id="cb9-25"><a href="#cb9-25" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>.projector <span class="op">=</span> nn.Sequential(</span>
<span id="cb9-26"><a href="#cb9-26" aria-hidden="true" tabindex="-1"></a>            nn.Linear(backbone_out_dim, backbone_out_dim),</span>
<span id="cb9-27"><a href="#cb9-27" aria-hidden="true" tabindex="-1"></a>            nn.BatchNorm1d(backbone_out_dim),</span>
<span id="cb9-28"><a href="#cb9-28" aria-hidden="true" tabindex="-1"></a>            nn.ReLU(inplace<span class="op">=</span><span class="va">True</span>),</span>
<span id="cb9-29"><a href="#cb9-29" aria-hidden="true" tabindex="-1"></a>            nn.Linear(backbone_out_dim, embed_dim),</span>
<span id="cb9-30"><a href="#cb9-30" aria-hidden="true" tabindex="-1"></a>            nn.BatchNorm1d(embed_dim)</span>
<span id="cb9-31"><a href="#cb9-31" aria-hidden="true" tabindex="-1"></a>        )</span>
<span id="cb9-32"><a href="#cb9-32" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb9-33"><a href="#cb9-33" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>.embed_dim <span class="op">=</span> embed_dim</span>
<span id="cb9-34"><a href="#cb9-34" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb9-35"><a href="#cb9-35" aria-hidden="true" tabindex="-1"></a>    <span class="kw">def</span> forward(<span class="va">self</span>, x: torch.Tensor) <span class="op">-&gt;</span> torch.Tensor:</span>
<span id="cb9-36"><a href="#cb9-36" aria-hidden="true" tabindex="-1"></a>        <span class="co"># x: (batch, 3, H, W)</span></span>
<span id="cb9-37"><a href="#cb9-37" aria-hidden="true" tabindex="-1"></a>        features <span class="op">=</span> <span class="va">self</span>.backbone(x)               <span class="co"># (batch, 2048, 1, 1)</span></span>
<span id="cb9-38"><a href="#cb9-38" aria-hidden="true" tabindex="-1"></a>        features <span class="op">=</span> features.flatten(start_dim<span class="op">=</span><span class="dv">1</span>)  <span class="co"># (batch, 2048)</span></span>
<span id="cb9-39"><a href="#cb9-39" aria-hidden="true" tabindex="-1"></a>        embeddings <span class="op">=</span> <span class="va">self</span>.projector(features)     <span class="co"># (batch, embed_dim)</span></span>
<span id="cb9-40"><a href="#cb9-40" aria-hidden="true" tabindex="-1"></a>        <span class="cf">return</span> F.normalize(embeddings, p<span class="op">=</span><span class="dv">2</span>, dim<span class="op">=</span><span class="dv">1</span>)  <span class="co"># L2 normalize</span></span></code></pre></div></div>
</div>
<div class="callout callout-style-default callout-tip callout-titled">
<div class="callout-header d-flex align-content-center">
<div class="callout-icon-container">
<i class="callout-icon"></i>

<div class="callout-title-container flex-fill">
Tip

<div class="callout-body-container callout-body">
<p><strong>Why L2 normalize?</strong> When vectors are unit-norm, inner product equals cosine similarity. This makes the <code>IP</code> metric in Milvus equivalent to cosine similarity, which is what most contrastive losses optimize.</p>

<section id="contrastive-loss-simclr-style" class="level4">
<h4 class="anchored" data-anchor-id="contrastive-loss-simclr-style">Contrastive Loss (SimCLR-style)</h4>
<div id="2e21cf8d" class="cell" data-execution_count="6">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb10"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb10-1"><a href="#cb10-1" aria-hidden="true" tabindex="-1"></a><span class="kw">class</span> NTXentLoss(nn.Module):</span>
<span id="cb10-2"><a href="#cb10-2" aria-hidden="true" tabindex="-1"></a>    <span class="co">"""</span></span>
<span id="cb10-3"><a href="#cb10-3" aria-hidden="true" tabindex="-1"></a><span class="co">    Normalized Temperature-scaled Cross Entropy Loss.</span></span>
<span id="cb10-4"><a href="#cb10-4" aria-hidden="true" tabindex="-1"></a><span class="co">    Used in SimCLR: https://arxiv.org/abs/2002.05709</span></span>
<span id="cb10-5"><a href="#cb10-5" aria-hidden="true" tabindex="-1"></a><span class="co">    """</span></span>
<span id="cb10-6"><a href="#cb10-6" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb10-7"><a href="#cb10-7" aria-hidden="true" tabindex="-1"></a>    <span class="kw">def</span> <span class="fu">__init__</span>(<span class="va">self</span>, temperature: <span class="bu">float</span> <span class="op">=</span> <span class="fl">0.07</span>):</span>
<span id="cb10-8"><a href="#cb10-8" aria-hidden="true" tabindex="-1"></a>        <span class="bu">super</span>().<span class="fu">__init__</span>()</span>
<span id="cb10-9"><a href="#cb10-9" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>.temperature <span class="op">=</span> temperature</span>
<span id="cb10-10"><a href="#cb10-10" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>.criterion <span class="op">=</span> nn.CrossEntropyLoss()</span>
<span id="cb10-11"><a href="#cb10-11" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb10-12"><a href="#cb10-12" aria-hidden="true" tabindex="-1"></a>    <span class="kw">def</span> forward(<span class="va">self</span>, z_i: torch.Tensor, z_j: torch.Tensor) <span class="op">-&gt;</span> torch.Tensor:</span>
<span id="cb10-13"><a href="#cb10-13" aria-hidden="true" tabindex="-1"></a>        <span class="co">"""</span></span>
<span id="cb10-14"><a href="#cb10-14" aria-hidden="true" tabindex="-1"></a><span class="co">        z_i, z_j: two augmented views of the same batch (both L2-normalized).</span></span>
<span id="cb10-15"><a href="#cb10-15" aria-hidden="true" tabindex="-1"></a><span class="co">        Shape: (batch_size, embed_dim)</span></span>
<span id="cb10-16"><a href="#cb10-16" aria-hidden="true" tabindex="-1"></a><span class="co">        """</span></span>
<span id="cb10-17"><a href="#cb10-17" aria-hidden="true" tabindex="-1"></a>        batch_size <span class="op">=</span> z_i.shape[<span class="dv">0</span>]</span>
<span id="cb10-18"><a href="#cb10-18" aria-hidden="true" tabindex="-1"></a>        z <span class="op">=</span> torch.cat([z_i, z_j], dim<span class="op">=</span><span class="dv">0</span>)          <span class="co"># (2B, D)</span></span>
<span id="cb10-19"><a href="#cb10-19" aria-hidden="true" tabindex="-1"></a>        sim <span class="op">=</span> torch.mm(z, z.T) <span class="op">/</span> <span class="va">self</span>.temperature  <span class="co"># (2B, 2B)</span></span>
<span id="cb10-20"><a href="#cb10-20" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb10-21"><a href="#cb10-21" aria-hidden="true" tabindex="-1"></a>        <span class="co"># Mask out self-similarity on the diagonal</span></span>
<span id="cb10-22"><a href="#cb10-22" aria-hidden="true" tabindex="-1"></a>        mask <span class="op">=</span> torch.eye(<span class="dv">2</span> <span class="op">*</span> batch_size, dtype<span class="op">=</span>torch.<span class="bu">bool</span>, device<span class="op">=</span>z.device)</span>
<span id="cb10-23"><a href="#cb10-23" aria-hidden="true" tabindex="-1"></a>        sim.masked_fill_(mask, <span class="bu">float</span>(<span class="st">'-inf'</span>))</span>
<span id="cb10-24"><a href="#cb10-24" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb10-25"><a href="#cb10-25" aria-hidden="true" tabindex="-1"></a>        <span class="co"># Positive pairs: (i, i+B) and (i+B, i)</span></span>
<span id="cb10-26"><a href="#cb10-26" aria-hidden="true" tabindex="-1"></a>        labels <span class="op">=</span> torch.cat([</span>
<span id="cb10-27"><a href="#cb10-27" aria-hidden="true" tabindex="-1"></a>            torch.arange(batch_size, <span class="dv">2</span> <span class="op">*</span> batch_size),</span>
<span id="cb10-28"><a href="#cb10-28" aria-hidden="true" tabindex="-1"></a>            torch.arange(<span class="dv">0</span>, batch_size)</span>
<span id="cb10-29"><a href="#cb10-29" aria-hidden="true" tabindex="-1"></a>        ]).to(z.device)</span>
<span id="cb10-30"><a href="#cb10-30" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb10-31"><a href="#cb10-31" aria-hidden="true" tabindex="-1"></a>        <span class="cf">return</span> <span class="va">self</span>.criterion(sim, labels)</span></code></pre></div></div>
</div>

<h4 class="anchored" data-anchor-id="triplet-loss-with-hard-negative-mining">Triplet Loss with Hard Negative Mining</h4>
<div id="ea03df68" class="cell" data-execution_count="7">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb11"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb11-1"><a href="#cb11-1" aria-hidden="true" tabindex="-1"></a><span class="kw">class</span> TripletLoss(nn.Module):</span>
<span id="cb11-2"><a href="#cb11-2" aria-hidden="true" tabindex="-1"></a>    <span class="co">"""</span></span>
<span id="cb11-3"><a href="#cb11-3" aria-hidden="true" tabindex="-1"></a><span class="co">    Triplet margin loss with online hard negative mining.</span></span>
<span id="cb11-4"><a href="#cb11-4" aria-hidden="true" tabindex="-1"></a><span class="co">    """</span></span>
<span id="cb11-5"><a href="#cb11-5" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb11-6"><a href="#cb11-6" aria-hidden="true" tabindex="-1"></a>    <span class="kw">def</span> <span class="fu">__init__</span>(<span class="va">self</span>, margin: <span class="bu">float</span> <span class="op">=</span> <span class="fl">0.3</span>):</span>
<span id="cb11-7"><a href="#cb11-7" aria-hidden="true" tabindex="-1"></a>        <span class="bu">super</span>().<span class="fu">__init__</span>()</span>
<span id="cb11-8"><a href="#cb11-8" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>.margin <span class="op">=</span> margin</span>
<span id="cb11-9"><a href="#cb11-9" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb11-10"><a href="#cb11-10" aria-hidden="true" tabindex="-1"></a>    <span class="kw">def</span> forward(</span>
<span id="cb11-11"><a href="#cb11-11" aria-hidden="true" tabindex="-1"></a>        <span class="va">self</span>,</span>
<span id="cb11-12"><a href="#cb11-12" aria-hidden="true" tabindex="-1"></a>        anchors: torch.Tensor,</span>
<span id="cb11-13"><a href="#cb11-13" aria-hidden="true" tabindex="-1"></a>        positives: torch.Tensor,</span>
<span id="cb11-14"><a href="#cb11-14" aria-hidden="true" tabindex="-1"></a>        negatives: torch.Tensor</span>
<span id="cb11-15"><a href="#cb11-15" aria-hidden="true" tabindex="-1"></a>    ) <span class="op">-&gt;</span> torch.Tensor:</span>
<span id="cb11-16"><a href="#cb11-16" aria-hidden="true" tabindex="-1"></a>        d_pos <span class="op">=</span> <span class="fl">1.0</span> <span class="op">-</span> (anchors <span class="op">*</span> positives).<span class="bu">sum</span>(dim<span class="op">=</span><span class="dv">1</span>)  <span class="co"># 1 - cosine sim</span></span>
<span id="cb11-17"><a href="#cb11-17" aria-hidden="true" tabindex="-1"></a>        d_neg <span class="op">=</span> <span class="fl">1.0</span> <span class="op">-</span> (anchors <span class="op">*</span> negatives).<span class="bu">sum</span>(dim<span class="op">=</span><span class="dv">1</span>)</span>
<span id="cb11-18"><a href="#cb11-18" aria-hidden="true" tabindex="-1"></a>        losses <span class="op">=</span> F.relu(d_pos <span class="op">-</span> d_neg <span class="op">+</span> <span class="va">self</span>.margin)</span>
<span id="cb11-19"><a href="#cb11-19" aria-hidden="true" tabindex="-1"></a>        <span class="cf">return</span> losses.mean()</span></code></pre></div></div>

<h2 class="anchored" data-anchor-id="generating-and-storing-embeddings-in-milvus" id="generating-and-storing-embeddings-in-milvus">Generating and Storing Embeddings in Milvus</h2>
<p>Before using Milvus during the training loop, you typically do an initial <strong>embedding pass</strong> over your dataset to populate the index.</p>
<section id="embedding-insertion-flow" class="level3">
<h3 class="anchored" data-anchor-id="embedding-insertion-flow" id="embedding-insertion-flow">Embedding Insertion Flow</h3>
<div class="cell" data-eval="true" data-layout-align="center">
<div class="cell-output-display">
<div>
<p></p><figure class="figure"><p></p>
<div>
<pre class="mermaid mermaid-js">flowchart TD
    DL[DataLoader batch] --&gt; FWD[Model forward pass]
    FWD --&gt; EMB[Embeddings tensor]
    EMB --&gt; BUF[Append to buffer]
    BUF --&gt; CHECK{Buffer size &gt;= 1000?}
    CHECK -- Yes --&gt; INS[client.insert]
    INS --&gt; CLR[Clear buffer]
    CLR --&gt; DL
    CHECK -- No --&gt; DL
    DL -- Exhausted --&gt; LAST{Buffer empty?}
    LAST -- No --&gt; INS2[client.insert remainder]
    INS2 --&gt; FLUSH[client.flush]
    LAST -- Yes --&gt; FLUSH
    FLUSH --&gt; DONE([Done])
</pre>

<section id="training-with-milvus-backed-retrieval" class="level2">
<h2 class="anchored" data-anchor-id="training-with-milvus-backed-retrieval" id="training-with-milvus-backed-retrieval">Training with Milvus-Backed Retrieval</h2>
<p>This section covers the core loop where Milvus actively participates in training by supplying <strong>hard negatives</strong> — examples that are semantically similar to an anchor but belong to a different class. Hard negatives are far more informative than random negatives for metric learning.</p>
<section id="the-hard-negative-mining-pattern" class="level3">
<h3 class="anchored" data-anchor-id="the-hard-negative-mining-pattern" id="the-hard-negative-mining-pattern">The Hard Negative Mining Pattern</h3>
<div class="cell" data-eval="true" data-layout-align="center">
<div class="cell-output-display">
<div>
<p></p><figure class="figure"><p></p>
<div>
<pre class="mermaid mermaid-js">flowchart LR
    A[Anchor embedding] --&gt; SEARCH["Milvus ANN search
    top-k results"]
    SEARCH --&gt; FILTER{Same label as anchor?}
    FILTER -- Yes --&gt; SKIP[Skip]
    FILTER -- No --&gt; SELECT[Select as hard negative]
    SKIP --&gt; FILTER
    SELECT --&gt; LOSS["Triplet Loss
    anchor, positive, hard_neg"]
</pre>
</div>
<p></p></figure><p></p>
</div>
</div>
</div>
<p>For each anchor in the batch:</p>
<ol type="1">
<li>Compute the anchor’s current embedding.</li>
<li>Query Milvus for the top-k most similar vectors that have a <strong>different</strong> label.</li>
<li>Use those returned vectors as negatives in the loss function.</li>
</ol>
<div id="d6e03cbc" class="cell" data-execution_count="10">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb14"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb14-1"><a href="#cb14-1" aria-hidden="true" tabindex="-1"></a><span class="kw">def</span> mine_hard_negatives(</span>
<span id="cb14-2"><a href="#cb14-2" aria-hidden="true" tabindex="-1"></a>    client: MilvusClient,</span>
<span id="cb14-3"><a href="#cb14-3" aria-hidden="true" tabindex="-1"></a>    collection_name: <span class="bu">str</span>,</span>
<span id="cb14-4"><a href="#cb14-4" aria-hidden="true" tabindex="-1"></a>    anchor_embeddings: np.ndarray,   <span class="co"># (B, D)</span></span>
<span id="cb14-5"><a href="#cb14-5" aria-hidden="true" tabindex="-1"></a>    anchor_labels: np.ndarray,        <span class="co"># (B,)</span></span>
<span id="cb14-6"><a href="#cb14-6" aria-hidden="true" tabindex="-1"></a>    top_k: <span class="bu">int</span> <span class="op">=</span> <span class="dv">64</span></span>
<span id="cb14-7"><a href="#cb14-7" aria-hidden="true" tabindex="-1"></a>) <span class="op">-&gt;</span> np.ndarray:</span>
<span id="cb14-8"><a href="#cb14-8" aria-hidden="true" tabindex="-1"></a>    <span class="co">"""</span></span>
<span id="cb14-9"><a href="#cb14-9" aria-hidden="true" tabindex="-1"></a><span class="co">    For each anchor, find the hardest negative (closest vector with a different label).</span></span>
<span id="cb14-10"><a href="#cb14-10" aria-hidden="true" tabindex="-1"></a><span class="co">    Returns hard_negatives of shape (B, D).</span></span>
<span id="cb14-11"><a href="#cb14-11" aria-hidden="true" tabindex="-1"></a><span class="co">    """</span></span>
<span id="cb14-12"><a href="#cb14-12" aria-hidden="true" tabindex="-1"></a>    hard_negatives <span class="op">=</span> np.zeros_like(anchor_embeddings)</span>
<span id="cb14-13"><a href="#cb14-13" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb14-14"><a href="#cb14-14" aria-hidden="true" tabindex="-1"></a>    results <span class="op">=</span> client.search(</span>
<span id="cb14-15"><a href="#cb14-15" aria-hidden="true" tabindex="-1"></a>        collection_name<span class="op">=</span>collection_name,</span>
<span id="cb14-16"><a href="#cb14-16" aria-hidden="true" tabindex="-1"></a>        data<span class="op">=</span>anchor_embeddings.tolist(),</span>
<span id="cb14-17"><a href="#cb14-17" aria-hidden="true" tabindex="-1"></a>        anns_field<span class="op">=</span><span class="st">"embedding"</span>,</span>
<span id="cb14-18"><a href="#cb14-18" aria-hidden="true" tabindex="-1"></a>        search_params<span class="op">=</span>{<span class="st">"metric_type"</span>: <span class="st">"IP"</span>, <span class="st">"params"</span>: {<span class="st">"ef"</span>: <span class="dv">128</span>}},</span>
<span id="cb14-19"><a href="#cb14-19" aria-hidden="true" tabindex="-1"></a>        limit<span class="op">=</span>top_k,</span>
<span id="cb14-20"><a href="#cb14-20" aria-hidden="true" tabindex="-1"></a>        output_fields<span class="op">=</span>[<span class="st">"embedding"</span>, <span class="st">"label"</span>],</span>
<span id="cb14-21"><a href="#cb14-21" aria-hidden="true" tabindex="-1"></a>        partition_names<span class="op">=</span>[<span class="st">"train"</span>]     <span class="co"># only search the training split</span></span>
<span id="cb14-22"><a href="#cb14-22" aria-hidden="true" tabindex="-1"></a>    )</span>
<span id="cb14-23"><a href="#cb14-23" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb14-24"><a href="#cb14-24" aria-hidden="true" tabindex="-1"></a>    <span class="cf">for</span> i, (query_results, anchor_label) <span class="kw">in</span> <span class="bu">enumerate</span>(<span class="bu">zip</span>(results, anchor_labels)):</span>
<span id="cb14-25"><a href="#cb14-25" aria-hidden="true" tabindex="-1"></a>        chosen <span class="op">=</span> <span class="va">None</span></span>
<span id="cb14-26"><a href="#cb14-26" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> hit <span class="kw">in</span> query_results:</span>
<span id="cb14-27"><a href="#cb14-27" aria-hidden="true" tabindex="-1"></a>            <span class="cf">if</span> hit[<span class="st">"entity"</span>][<span class="st">"label"</span>] <span class="op">!=</span> <span class="bu">int</span>(anchor_label):</span>
<span id="cb14-28"><a href="#cb14-28" aria-hidden="true" tabindex="-1"></a>                chosen <span class="op">=</span> np.array(hit[<span class="st">"entity"</span>][<span class="st">"embedding"</span>], dtype<span class="op">=</span>np.float32)</span>
<span id="cb14-29"><a href="#cb14-29" aria-hidden="true" tabindex="-1"></a>                <span class="cf">break</span></span>
<span id="cb14-30"><a href="#cb14-30" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb14-31"><a href="#cb14-31" aria-hidden="true" tabindex="-1"></a>        <span class="cf">if</span> chosen <span class="kw">is</span> <span class="va">None</span>:</span>
<span id="cb14-32"><a href="#cb14-32" aria-hidden="true" tabindex="-1"></a>            <span class="co"># Fallback: use the farthest result in the returned set</span></span>
<span id="cb14-33"><a href="#cb14-33" aria-hidden="true" tabindex="-1"></a>            chosen <span class="op">=</span> np.array(query_results[<span class="op">-</span><span class="dv">1</span>][<span class="st">"entity"</span>][<span class="st">"embedding"</span>], dtype<span class="op">=</span>np.float32)</span>
<span id="cb14-34"><a href="#cb14-34" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb14-35"><a href="#cb14-35" aria-hidden="true" tabindex="-1"></a>        hard_negatives[i] <span class="op">=</span> chosen</span>
<span id="cb14-36"><a href="#cb14-36" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb14-37"><a href="#cb14-37" aria-hidden="true" tabindex="-1"></a>    <span class="cf">return</span> hard_negatives</span></code></pre></div></div>

<span id="cb15-28"><a href="#cb15-28" aria-hidden="true" tabindex="-1"></a>            labels_np <span class="op">=</span> labels.numpy()</span>
<span id="cb15-29"><a href="#cb15-29" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-30"><a href="#cb15-30" aria-hidden="true" tabindex="-1"></a>            <span class="co"># --- Forward pass ---</span></span>
<span id="cb15-31"><a href="#cb15-31" aria-hidden="true" tabindex="-1"></a>            embeddings <span class="op">=</span> model(images)           <span class="co"># (B, 512), L2-normalized</span></span>
<span id="cb15-32"><a href="#cb15-32" aria-hidden="true" tabindex="-1"></a>            emb_np <span class="op">=</span> embeddings.detach().cpu().numpy()</span>
<span id="cb15-33"><a href="#cb15-33" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-34"><a href="#cb15-34" aria-hidden="true" tabindex="-1"></a>            <span class="co"># --- Mine hard negatives from Milvus ---</span></span>
<span id="cb15-35"><a href="#cb15-35" aria-hidden="true" tabindex="-1"></a>            hard_neg_np <span class="op">=</span> mine_hard_negatives(</span>
<span id="cb15-36"><a href="#cb15-36" aria-hidden="true" tabindex="-1"></a>                client, collection_name, emb_np, labels_np</span>
<span id="cb15-37"><a href="#cb15-37" aria-hidden="true" tabindex="-1"></a>            )</span>
<span id="cb15-38"><a href="#cb15-38" aria-hidden="true" tabindex="-1"></a>            hard_neg <span class="op">=</span> torch.tensor(hard_neg_np, dtype<span class="op">=</span>torch.float32, device<span class="op">=</span>device)</span>
<span id="cb15-39"><a href="#cb15-39" aria-hidden="true" tabindex="-1"></a>            hard_neg <span class="op">=</span> F.normalize(hard_neg, p<span class="op">=</span><span class="dv">2</span>, dim<span class="op">=</span><span class="dv">1</span>)</span>
<span id="cb15-40"><a href="#cb15-40" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-41"><a href="#cb15-41" aria-hidden="true" tabindex="-1"></a>            <span class="co"># --- Construct positives (augmented duplicate in same batch) ---</span></span>
<span id="cb15-42"><a href="#cb15-42" aria-hidden="true" tabindex="-1"></a>            <span class="co"># Shuffle within same-label groups to get positives</span></span>
<span id="cb15-43"><a href="#cb15-43" aria-hidden="true" tabindex="-1"></a>            positive_idx <span class="op">=</span> torch.zeros(<span class="bu">len</span>(labels), dtype<span class="op">=</span>torch.<span class="bu">long</span>)</span>
<span id="cb15-44"><a href="#cb15-44" aria-hidden="true" tabindex="-1"></a>            <span class="cf">for</span> lbl <span class="kw">in</span> labels.unique():</span>
<span id="cb15-45"><a href="#cb15-45" aria-hidden="true" tabindex="-1"></a>                mask <span class="op">=</span> (labels <span class="op">==</span> lbl).nonzero(as_tuple<span class="op">=</span><span class="va">True</span>)[<span class="dv">0</span>]</span>
<span id="cb15-46"><a href="#cb15-46" aria-hidden="true" tabindex="-1"></a>                <span class="cf">if</span> <span class="bu">len</span>(mask) <span class="op">&gt;=</span> <span class="dv">2</span>:</span>
<span id="cb15-47"><a href="#cb15-47" aria-hidden="true" tabindex="-1"></a>                    shuffled <span class="op">=</span> mask[torch.randperm(<span class="bu">len</span>(mask))]</span>
<span id="cb15-48"><a href="#cb15-48" aria-hidden="true" tabindex="-1"></a>                    positive_idx[mask] <span class="op">=</span> shuffled</span>
<span id="cb15-49"><a href="#cb15-49" aria-hidden="true" tabindex="-1"></a>                <span class="cf">else</span>:</span>
<span id="cb15-50"><a href="#cb15-50" aria-hidden="true" tabindex="-1"></a>                    positive_idx[mask] <span class="op">=</span> mask  <span class="co"># fallback: self-positive (degenerate)</span></span>
<span id="cb15-51"><a href="#cb15-51" aria-hidden="true" tabindex="-1"></a>            positives <span class="op">=</span> embeddings[positive_idx]</span>
<span id="cb15-52"><a href="#cb15-52" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-53"><a href="#cb15-53" aria-hidden="true" tabindex="-1"></a>            <span class="co"># --- Compute loss ---</span></span>
<span id="cb15-54"><a href="#cb15-54" aria-hidden="true" tabindex="-1"></a>            loss <span class="op">=</span> triplet_loss(embeddings, positives, hard_neg)</span>
<span id="cb15-55"><a href="#cb15-55" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-56"><a href="#cb15-56" aria-hidden="true" tabindex="-1"></a>            <span class="co"># --- Backward pass ---</span></span>
<span id="cb15-57"><a href="#cb15-57" aria-hidden="true" tabindex="-1"></a>            optimizer.zero_grad()</span>
<span id="cb15-58"><a href="#cb15-58" aria-hidden="true" tabindex="-1"></a>            loss.backward()</span>
<span id="cb15-59"><a href="#cb15-59" aria-hidden="true" tabindex="-1"></a>            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm<span class="op">=</span><span class="fl">1.0</span>)</span>
<span id="cb15-60"><a href="#cb15-60" aria-hidden="true" tabindex="-1"></a>            optimizer.step()</span>
<span id="cb15-61"><a href="#cb15-61" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-62"><a href="#cb15-62" aria-hidden="true" tabindex="-1"></a>            epoch_loss <span class="op">+=</span> loss.item()</span>
<span id="cb15-63"><a href="#cb15-63" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-64"><a href="#cb15-64" aria-hidden="true" tabindex="-1"></a>        scheduler.step()</span>
<span id="cb15-65"><a href="#cb15-65" aria-hidden="true" tabindex="-1"></a>        avg_loss <span class="op">=</span> epoch_loss <span class="op">/</span> <span class="bu">len</span>(train_loader)</span>
<span id="cb15-66"><a href="#cb15-66" aria-hidden="true" tabindex="-1"></a>        <span class="bu">print</span>(<span class="ss">f"Epoch </span><span class="sc">{</span>epoch<span class="sc">:3d}</span><span class="ss"> | Loss: </span><span class="sc">{</span>avg_loss<span class="sc">:.4f}</span><span class="ss"> | LR: </span><span class="sc">{</span>scheduler<span class="sc">.</span>get_last_lr()[<span class="dv">0</span>]<span class="sc">:.2e}</span><span class="ss">"</span>)</span>
<span id="cb15-67"><a href="#cb15-67" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-68"><a href="#cb15-68" aria-hidden="true" tabindex="-1"></a>        <span class="co"># --- Periodically re-index to keep Milvus up-to-date ---</span></span>
<span id="cb15-69"><a href="#cb15-69" aria-hidden="true" tabindex="-1"></a>        <span class="cf">if</span> epoch <span class="op">%</span> reindex_every <span class="op">==</span> <span class="dv">0</span>:</span>
<span id="cb15-70"><a href="#cb15-70" aria-hidden="true" tabindex="-1"></a>            <span class="bu">print</span>(<span class="ss">f"Re-indexing after epoch </span><span class="sc">{</span>epoch<span class="sc">}</span><span class="ss">..."</span>)</span>
<span id="cb15-71"><a href="#cb15-71" aria-hidden="true" tabindex="-1"></a>            client.delete(collection_name, <span class="bu">filter</span><span class="op">=</span><span class="ss">f'split == "train"'</span>)</span>
<span id="cb15-72"><a href="#cb15-72" aria-hidden="true" tabindex="-1"></a>            embed_dataset(model, train_loader, device, <span class="st">"train"</span>, client, collection_name)</span>
<span id="cb15-73"><a href="#cb15-73" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb15-74"><a href="#cb15-74" aria-hidden="true" tabindex="-1"></a>        <span class="co"># --- Save checkpoint ---</span></span>
<span id="cb15-75"><a href="#cb15-75" aria-hidden="true" tabindex="-1"></a>        ckpt_path <span class="op">=</span> os.path.join(checkpoint_dir, <span class="ss">f"encoder_epoch</span><span class="sc">{</span>epoch<span class="sc">:03d}</span><span class="ss">.pt"</span>)</span>
<span id="cb15-76"><a href="#cb15-76" aria-hidden="true" tabindex="-1"></a>        torch.save({</span>
<span id="cb15-77"><a href="#cb15-77" aria-hidden="true" tabindex="-1"></a>            <span class="st">"epoch"</span>: epoch,</span>
<span id="cb15-78"><a href="#cb15-78" aria-hidden="true" tabindex="-1"></a>            <span class="st">"model_state_dict"</span>: model.state_dict(),</span>
<span id="cb15-79"><a href="#cb15-79" aria-hidden="true" tabindex="-1"></a>            <span class="st">"optimizer_state_dict"</span>: optimizer.state_dict(),</span>
<span id="cb15-80"><a href="#cb15-80" aria-hidden="true" tabindex="-1"></a>            <span class="st">"loss"</span>: avg_loss</span>
<span id="cb15-81"><a href="#cb15-81" aria-hidden="true" tabindex="-1"></a>        }, ckpt_path)</span></code></pre></div></div>
</div>

<h3 class="anchored" data-anchor-id="k-nn-evaluation-flow" id="k-nn-evaluation-flow">k-NN Evaluation Flow</h3>
<div class="cell" data-eval="true" data-layout-align="center">
<div class="cell-output-display">
<div>
<p></p><figure class="figure"><p></p>
<div>
<pre class="mermaid mermaid-js">flowchart TD
    VAL[Val image] --&gt; ENC[Encode with model]
    ENC --&gt; SEARCH[Search Milvus train split, top-k neighbors]
    SEARCH --&gt; VOTE[Majority vote on labels]
    VOTE --&gt; PRED[Predicted class]
    PRED --&gt; CMP{Matches true label?}
    CMP -- Yes --&gt; CORRECT[correct += 1]
    CMP -- No --&gt; WRONG[Skip]
    CORRECT --&gt; TOTAL[total += 1]
    WRONG --&gt; TOTAL
    TOTAL --&gt; NEXT{More images?}
    NEXT -- Yes --&gt; VAL
    NEXT -- No --&gt; ACC["accuracy = correct / total"]
</pre>

<section id="updating-the-index-during-training" class="level2">
<h2 class="anchored" data-anchor-id="updating-the-index-during-training" id="updating-the-index-during-training">Updating the Index During Training</h2>
<p>Keeping the Milvus index synchronized with a changing model is one of the more nuanced aspects of this architecture. Here are three strategies:</p>
<section id="strategy-comparison" class="level3">
<h3 class="anchored" data-anchor-id="strategy-comparison" id="strategy-comparison">Strategy Comparison</h3>
<div class="cell" data-eval="true" data-layout-align="center">
<div class="cell-output-display">
<div>
<p></p><figure class="figure"><p></p>
<div>
<pre class="mermaid mermaid-js">flowchart TD
    START([Choose re-index strategy]) --&gt; Q1{Dataset size}
    Q1 -- "&lt; 500K" --&gt; FULL[Strategy 1: Full Re-Index - Delete all, re-embed all]
    Q1 -- "500K - 10M" --&gt; INCR[Strategy 2: Incremental Update - Re-embed high-loss samples only]
    Q1 -- "&gt; 10M" --&gt; MOM[Strategy 3: Momentum Encoder - EMA model writes to Milvus continuously]
    FULL --&gt; SIMPLE[Simple and consistent - Expensive per epoch]
    INCR --&gt; MEDIUM[Efficient and targeted - Requires loss tracking]
    MOM --&gt; STABLE[Stable negatives - Most complex to implement]
</pre>
</div>
<p></p></figure><p></p>
</div>
</div>
</div>
</section>
<section id="strategy-1-periodic-full-re-index-simple" class="level3">
<h3 class="anchored" data-anchor-id="strategy-1-periodic-full-re-index-simple" id="strategy-1-periodic-full-re-index-simple">Strategy 1: Periodic Full Re-Index (Simple)</h3>
<p>As shown in the training loop above, delete all training embeddings every N epochs and recompute them. This is simple but expensive if your dataset is large.</p>
<div id="247bb882" class="cell" data-execution_count="14">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb18"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb18-1"><a href="#cb18-1" aria-hidden="true" tabindex="-1"></a><span class="kw">def</span> full_reindex(model, loader, device, client, collection_name, split<span class="op">=</span><span class="st">"train"</span>):</span>
<span id="cb18-2"><a href="#cb18-2" aria-hidden="true" tabindex="-1"></a>    client.delete(collection_name, <span class="bu">filter</span><span class="op">=</span><span class="ss">f'split == "</span><span class="sc">{</span>split<span class="sc">}</span><span class="ss">"'</span>)</span>
<span id="cb18-3"><a href="#cb18-3" aria-hidden="true" tabindex="-1"></a>    client.flush(collection_name)</span>
<span id="cb18-4"><a href="#cb18-4" aria-hidden="true" tabindex="-1"></a>    embed_dataset(model, loader, device, split, client, collection_name)</span></code></pre></div></div>
</div>
<p><strong>When to use:</strong> Datasets under ~500K samples, re-index every 5–10 epochs.</p>
</section>
<section id="strategy-2-incremental-update-efficient" class="level3">
<h3 class="anchored" data-anchor-id="strategy-2-incremental-update-efficient" id="strategy-2-incremental-update-efficient">Strategy 2: Incremental Update (Efficient)</h3>
<p>Maintain a mapping from dataset index to Milvus ID. After each epoch, re-embed only items whose loss exceeded a threshold, then call <code>upsert</code>:</p>
<div id="0bd5b03b" class="cell" data-execution_count="15">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb19"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb19-1"><a href="#cb19-1" aria-hidden="true" tabindex="-1"></a><span class="kw">def</span> incremental_update(</span>
<span id="cb19-2"><a href="#cb19-2" aria-hidden="true" tabindex="-1"></a>    high_loss_indices: <span class="bu">list</span>,</span>
<span id="cb19-3"><a href="#cb19-3" aria-hidden="true" tabindex="-1"></a>    dataset,</span>
<span id="cb19-4"><a href="#cb19-4" aria-hidden="true" tabindex="-1"></a>    model: nn.Module,</span>
<span id="cb19-5"><a href="#cb19-5" aria-hidden="true" tabindex="-1"></a>    id_map: <span class="bu">dict</span>,       <span class="co"># dataset_index -&gt; milvus_id</span></span>
<span id="cb19-6"><a href="#cb19-6" aria-hidden="true" tabindex="-1"></a>    client: MilvusClient,</span>
<span id="cb19-7"><a href="#cb19-7" aria-hidden="true" tabindex="-1"></a>    collection_name: <span class="bu">str</span>,</span>
<span id="cb19-8"><a href="#cb19-8" aria-hidden="true" tabindex="-1"></a>    device: torch.device</span>
<span id="cb19-9"><a href="#cb19-9" aria-hidden="true" tabindex="-1"></a>):</span>
<span id="cb19-10"><a href="#cb19-10" aria-hidden="true" tabindex="-1"></a>    model.<span class="bu">eval</span>()</span>
<span id="cb19-11"><a href="#cb19-11" aria-hidden="true" tabindex="-1"></a>    <span class="cf">with</span> torch.no_grad():</span>
<span id="cb19-12"><a href="#cb19-12" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> idx <span class="kw">in</span> high_loss_indices:</span>
<span id="cb19-13"><a href="#cb19-13" aria-hidden="true" tabindex="-1"></a>            image, label <span class="op">=</span> dataset[idx]</span>
<span id="cb19-14"><a href="#cb19-14" aria-hidden="true" tabindex="-1"></a>            tensor <span class="op">=</span> image.unsqueeze(<span class="dv">0</span>).to(device)</span>
<span id="cb19-15"><a href="#cb19-15" aria-hidden="true" tabindex="-1"></a>            emb <span class="op">=</span> model(tensor).squeeze(<span class="dv">0</span>).cpu().numpy()</span>
<span id="cb19-16"><a href="#cb19-16" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb19-17"><a href="#cb19-17" aria-hidden="true" tabindex="-1"></a>            client.upsert(</span>
<span id="cb19-18"><a href="#cb19-18" aria-hidden="true" tabindex="-1"></a>                collection_name<span class="op">=</span>collection_name,</span>
<span id="cb19-19"><a href="#cb19-19" aria-hidden="true" tabindex="-1"></a>                data<span class="op">=</span>[{</span>
<span id="cb19-20"><a href="#cb19-20" aria-hidden="true" tabindex="-1"></a>                    <span class="st">"id"</span>: id_map[idx],</span>
<span id="cb19-21"><a href="#cb19-21" aria-hidden="true" tabindex="-1"></a>                    <span class="st">"embedding"</span>: emb.tolist(),</span>
<span id="cb19-22"><a href="#cb19-22" aria-hidden="true" tabindex="-1"></a>                    <span class="st">"label"</span>: <span class="bu">int</span>(label),</span>
<span id="cb19-23"><a href="#cb19-23" aria-hidden="true" tabindex="-1"></a>                    <span class="st">"split"</span>: <span class="st">"train"</span></span>
<span id="cb19-24"><a href="#cb19-24" aria-hidden="true" tabindex="-1"></a>                }]</span>
<span id="cb19-25"><a href="#cb19-25" aria-hidden="true" tabindex="-1"></a>            )</span>
<span id="cb19-26"><a href="#cb19-26" aria-hidden="true" tabindex="-1"></a>    client.flush(collection_name)</span></code></pre></div></div>
</div>
<p><strong>When to use:</strong> Large datasets where full re-index is too costly; track per-sample loss and only update stale embeddings.</p>
</section>
<section id="strategy-3-momentum-encoder-advanced" class="level3">
<h3 class="anchored" data-anchor-id="strategy-3-momentum-encoder-advanced" id="strategy-3-momentum-encoder-advanced">Strategy 3: Momentum Encoder (Advanced)</h3>
<p>Inspired by MoCo (He et al., 2020), maintain two model weights:</p>
<ul>
<li><code>model</code> (online encoder): updated by gradients each step.</li>
<li><code>momentum_model</code> (key encoder): updated as an exponential moving average of <code>model</code>.</li>
</ul>
<p>Milvus stores embeddings produced by <code>momentum_model</code>, which provides more stable negatives:</p>
<div id="8b925114" class="cell" data-execution_count="16">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb20"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb20-1"><a href="#cb20-1" aria-hidden="true" tabindex="-1"></a><span class="im">import</span> copy</span>
<span id="cb20-2"><a href="#cb20-2" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb20-3"><a href="#cb20-3" aria-hidden="true" tabindex="-1"></a>momentum_model <span class="op">=</span> copy.deepcopy(model)</span>
<span id="cb20-4"><a href="#cb20-4" aria-hidden="true" tabindex="-1"></a>momentum_model.<span class="bu">eval</span>()</span>
<span id="cb20-5"><a href="#cb20-5" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb20-6"><a href="#cb20-6" aria-hidden="true" tabindex="-1"></a>MOMENTUM <span class="op">=</span> <span class="fl">0.999</span></span>
<span id="cb20-7"><a href="#cb20-7" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb20-8"><a href="#cb20-8" aria-hidden="true" tabindex="-1"></a><span class="kw">def</span> update_momentum_encoder(model, momentum_model, m<span class="op">=</span>MOMENTUM):</span>
<span id="cb20-9"><a href="#cb20-9" aria-hidden="true" tabindex="-1"></a>    <span class="cf">with</span> torch.no_grad():</span>
<span id="cb20-10"><a href="#cb20-10" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> param_q, param_k <span class="kw">in</span> <span class="bu">zip</span>(model.parameters(), momentum_model.parameters()):</span>
<span id="cb20-11"><a href="#cb20-11" aria-hidden="true" tabindex="-1"></a>            param_k.data <span class="op">=</span> m <span class="op">*</span> param_k.data <span class="op">+</span> (<span class="fl">1.0</span> <span class="op">-</span> m) <span class="op">*</span> param_q.data</span></code></pre></div></div>
</div>
<p>Call <code>update_momentum_encoder</code> at the end of each optimizer step. Use <code>momentum_model</code> for generating the embeddings inserted into Milvus.</p>

<section id="common-pitfalls-and-debugging" class="level2">
<h2 class="anchored" data-anchor-id="common-pitfalls-and-debugging" id="common-pitfalls-and-debugging">Common Pitfalls and Debugging</h2>
<section id="embedding-dimensions-dont-match" class="level3">
<h3 class="anchored" data-anchor-id="embedding-dimensions-dont-match" id="embedding-dimensions-dont-match">Embedding Dimensions Don’t Match</h3>
<p><strong>Symptom:</strong> <code>pymilvus.exceptions.MilvusException: ... vector size 768 not match collection dim 512</code></p>
<p><strong>Fix:</strong> Ensure your model’s <code>embed_dim</code> matches the <code>dim</code> set in the collection schema. If you change the architecture, drop and recreate the collection.</p>
<div id="72db3597" class="cell" data-execution_count="20">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb24"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb24-1"><a href="#cb24-1" aria-hidden="true" tabindex="-1"></a><span class="co"># Always print the shape of a sample embedding before inserting</span></span>
<span id="cb24-2"><a href="#cb24-2" aria-hidden="true" tabindex="-1"></a>sample <span class="op">=</span> model(torch.randn(<span class="dv">1</span>, <span class="dv">3</span>, <span class="dv">224</span>, <span class="dv">224</span>).to(device))</span>

</section>
<section id="vectors-are-not-normalized" class="level3">
<h3 class="anchored" data-anchor-id="vectors-are-not-normalized" id="vectors-are-not-normalized">Vectors Are Not Normalized</h3>
<p><strong>Symptom:</strong> Cosine similarity scores &gt; 1 or nonsensical; loss doesn’t converge.</p>
<p><strong>Fix:</strong> Always call <code>F.normalize(embeddings, p=2, dim=1)</code> before inserting into Milvus when using <code>IP</code> metric. Verify:</p>
<div id="7078cae2" class="cell" data-execution_count="21">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb25"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb25-1"><a href="#cb25-1" aria-hidden="true" tabindex="-1"></a>norms <span class="op">=</span> embeddings.norm(dim<span class="op">=</span><span class="dv">1</span>)</span>
<span id="cb25-2"><a href="#cb25-2" aria-hidden="true" tabindex="-1"></a><span class="cf">assert</span> torch.allclose(norms, torch.ones_like(norms), atol<span class="op">=</span><span class="fl">1e-5</span>), <span class="op">\</span></span>

</div>
</section>
<section id="milvus-returns-stale-embeddings" class="level3">
<h3 class="anchored" data-anchor-id="milvus-returns-stale-embeddings" id="milvus-returns-stale-embeddings">Milvus Returns Stale Embeddings</h3>
<p><strong>Symptom:</strong> Hard negatives seem too easy despite many training epochs.</p>
<p><strong>Fix:</strong> Either increase re-indexing frequency or use the momentum encoder strategy. Also call <code>client.flush()</code> after inserting to ensure newly inserted vectors are searchable.</p>
</section>
<section id="out-of-memory-during-re-index" class="level3">
<h3 class="anchored" data-anchor-id="out-of-memory-during-re-index" id="out-of-memory-during-re-index">Out of Memory During Re-Index</h3>
<p><strong>Symptom:</strong> CUDA OOM during embedding pass.</p>
<p><strong>Fix:</strong> Use <code>torch.no_grad()</code>, reduce <code>batch_size</code> during the embedding pass, and use <code>pin_memory=True</code> + <code>non_blocking=True</code> for efficient GPU transfers.</p>
</section>
<section id="index-not-loaded" class="level3">
<h3 class="anchored" data-anchor-id="index-not-loaded" id="index-not-loaded">Index Not Loaded</h3>
<p><strong>Symptom:</strong> <code>MilvusException: collection not loaded</code></p>
<p><strong>Fix:</strong> After creating a collection or reconnecting, always call:</p>
<div id="baf23325" class="cell" data-execution_count="22">
<div class="code-copy-outer-scaffold"><div class="sourceCode cell-code" id="cb26"><pre class="sourceCode python code-with-copy"><code class="sourceCode python"><span id="cb26-1"><a href="#cb26-1" aria-hidden="true" tabindex="-1"></a>client.load_collection(COLLECTION_NAME)</span></code></pre></div></div>
</div>
<p>Collections are loaded lazily into memory. In production, use Milvus’s <code>autoLoad</code> configuration to load on startup.</p>
<hr>
</section>

<span id="cb27-104"><a href="#cb27-104" aria-hidden="true" tabindex="-1"></a>    )</span>
<span id="cb27-105"><a href="#cb27-105" aria-hidden="true" tabindex="-1"></a>    negs <span class="op">=</span> np.zeros_like(embs)</span>
<span id="cb27-106"><a href="#cb27-106" aria-hidden="true" tabindex="-1"></a>    <span class="cf">for</span> i, (hits, lbl) <span class="kw">in</span> <span class="bu">enumerate</span>(<span class="bu">zip</span>(results, labels)):</span>
<span id="cb27-107"><a href="#cb27-107" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> h <span class="kw">in</span> hits:</span>
<span id="cb27-108"><a href="#cb27-108" aria-hidden="true" tabindex="-1"></a>            <span class="cf">if</span> h[<span class="st">"entity"</span>][<span class="st">"label"</span>] <span class="op">!=</span> <span class="bu">int</span>(lbl):</span>
<span id="cb27-109"><a href="#cb27-109" aria-hidden="true" tabindex="-1"></a>                negs[i] <span class="op">=</span> np.array(h[<span class="st">"entity"</span>][<span class="st">"embedding"</span>], dtype<span class="op">=</span>np.float32)</span>
<span id="cb27-110"><a href="#cb27-110" aria-hidden="true" tabindex="-1"></a>                <span class="cf">break</span></span>
<span id="cb27-111"><a href="#cb27-111" aria-hidden="true" tabindex="-1"></a>        <span class="cf">else</span>:</span>
<span id="cb27-112"><a href="#cb27-112" aria-hidden="true" tabindex="-1"></a>            negs[i] <span class="op">=</span> np.array(hits[<span class="op">-</span><span class="dv">1</span>][<span class="st">"entity"</span>][<span class="st">"embedding"</span>], dtype<span class="op">=</span>np.float32)</span>
<span id="cb27-113"><a href="#cb27-113" aria-hidden="true" tabindex="-1"></a>    <span class="cf">return</span> negs</span>
<span id="cb27-114"><a href="#cb27-114" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-115"><a href="#cb27-115" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-116"><a href="#cb27-116" aria-hidden="true" tabindex="-1"></a><span class="co"># -- Main --</span></span>
<span id="cb27-117"><a href="#cb27-117" aria-hidden="true" tabindex="-1"></a><span class="kw">def</span> main():</span>
<span id="cb27-118"><a href="#cb27-118" aria-hidden="true" tabindex="-1"></a>    <span class="co"># Data</span></span>
<span id="cb27-119"><a href="#cb27-119" aria-hidden="true" tabindex="-1"></a>    train_tf <span class="op">=</span> T.Compose([</span>
<span id="cb27-120"><a href="#cb27-120" aria-hidden="true" tabindex="-1"></a>        T.RandomResizedCrop(<span class="dv">224</span>), T.RandomHorizontalFlip(),</span>
<span id="cb27-121"><a href="#cb27-121" aria-hidden="true" tabindex="-1"></a>        T.ColorJitter(<span class="fl">0.4</span>, <span class="fl">0.4</span>, <span class="fl">0.4</span>, <span class="fl">0.1</span>),</span>
<span id="cb27-122"><a href="#cb27-122" aria-hidden="true" tabindex="-1"></a>        T.ToTensor(), T.Normalize([<span class="fl">0.485</span>, <span class="fl">0.456</span>, <span class="fl">0.406</span>], [<span class="fl">0.229</span>, <span class="fl">0.224</span>, <span class="fl">0.225</span>])</span>
<span id="cb27-123"><a href="#cb27-123" aria-hidden="true" tabindex="-1"></a>    ])</span>
<span id="cb27-124"><a href="#cb27-124" aria-hidden="true" tabindex="-1"></a>    val_tf <span class="op">=</span> T.Compose([</span>
<span id="cb27-125"><a href="#cb27-125" aria-hidden="true" tabindex="-1"></a>        T.Resize(<span class="dv">256</span>), T.CenterCrop(<span class="dv">224</span>),</span>
<span id="cb27-126"><a href="#cb27-126" aria-hidden="true" tabindex="-1"></a>        T.ToTensor(), T.Normalize([<span class="fl">0.485</span>, <span class="fl">0.456</span>, <span class="fl">0.406</span>], [<span class="fl">0.229</span>, <span class="fl">0.224</span>, <span class="fl">0.225</span>])</span>
<span id="cb27-127"><a href="#cb27-127" aria-hidden="true" tabindex="-1"></a>    ])</span>
<span id="cb27-128"><a href="#cb27-128" aria-hidden="true" tabindex="-1"></a>    train_ds <span class="op">=</span> ImageFolder(<span class="ss">f"</span><span class="sc">{</span>DATA_ROOT<span class="sc">}</span><span class="ss">/train"</span>, transform<span class="op">=</span>train_tf)</span>
<span id="cb27-129"><a href="#cb27-129" aria-hidden="true" tabindex="-1"></a>    val_ds   <span class="op">=</span> ImageFolder(<span class="ss">f"</span><span class="sc">{</span>DATA_ROOT<span class="sc">}</span><span class="ss">/val"</span>,   transform<span class="op">=</span>val_tf)</span>
<span id="cb27-130"><a href="#cb27-130" aria-hidden="true" tabindex="-1"></a>    train_dl <span class="op">=</span> DataLoader(train_ds, BATCH_SIZE, shuffle<span class="op">=</span><span class="va">True</span>,  num_workers<span class="op">=</span><span class="dv">4</span>, pin_memory<span class="op">=</span><span class="va">True</span>, drop_last<span class="op">=</span><span class="va">True</span>)</span>
<span id="cb27-131"><a href="#cb27-131" aria-hidden="true" tabindex="-1"></a>    val_dl   <span class="op">=</span> DataLoader(val_ds,  <span class="dv">256</span>,         shuffle<span class="op">=</span><span class="va">False</span>, num_workers<span class="op">=</span><span class="dv">4</span>, pin_memory<span class="op">=</span><span class="va">True</span>)</span>
<span id="cb27-132"><a href="#cb27-132" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-133"><a href="#cb27-133" aria-hidden="true" tabindex="-1"></a>    <span class="co"># Model + Milvus</span></span>
<span id="cb27-134"><a href="#cb27-134" aria-hidden="true" tabindex="-1"></a>    model  <span class="op">=</span> ImageEncoder(EMBED_DIM).to(DEVICE)</span>
<span id="cb27-135"><a href="#cb27-135" aria-hidden="true" tabindex="-1"></a>    client <span class="op">=</span> setup_milvus(MILVUS_URI, COLLECTION_NAME, EMBED_DIM)</span>
<span id="cb27-136"><a href="#cb27-136" aria-hidden="true" tabindex="-1"></a>    embed_and_store(model, train_dl, <span class="st">"train"</span>, client, COLLECTION_NAME, DEVICE)</span>
<span id="cb27-137"><a href="#cb27-137" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-138"><a href="#cb27-138" aria-hidden="true" tabindex="-1"></a>    <span class="co"># Optimizer</span></span>
<span id="cb27-139"><a href="#cb27-139" aria-hidden="true" tabindex="-1"></a>    opt     <span class="op">=</span> AdamW(model.parameters(), lr<span class="op">=</span>LEARNING_RATE, weight_decay<span class="op">=</span>WEIGHT_DECAY)</span>
<span id="cb27-140"><a href="#cb27-140" aria-hidden="true" tabindex="-1"></a>    sched   <span class="op">=</span> CosineAnnealingLR(opt, T_max<span class="op">=</span>NUM_EPOCHS)</span>
<span id="cb27-141"><a href="#cb27-141" aria-hidden="true" tabindex="-1"></a>    loss_fn <span class="op">=</span> TripletLoss(TRIPLET_MARGIN)</span>
<span id="cb27-142"><a href="#cb27-142" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-143"><a href="#cb27-143" aria-hidden="true" tabindex="-1"></a>    <span class="cf">for</span> epoch <span class="kw">in</span> <span class="bu">range</span>(<span class="dv">1</span>, NUM_EPOCHS <span class="op">+</span> <span class="dv">1</span>):</span>
<span id="cb27-144"><a href="#cb27-144" aria-hidden="true" tabindex="-1"></a>        model.train()</span>
<span id="cb27-145"><a href="#cb27-145" aria-hidden="true" tabindex="-1"></a>        total_loss <span class="op">=</span> <span class="fl">0.0</span></span>
<span id="cb27-146"><a href="#cb27-146" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-147"><a href="#cb27-147" aria-hidden="true" tabindex="-1"></a>        <span class="cf">for</span> imgs, lbls <span class="kw">in</span> tqdm(train_dl, desc<span class="op">=</span><span class="ss">f"Epoch </span><span class="sc">{</span>epoch<span class="sc">:3d}</span><span class="ss">"</span>):</span>
<span id="cb27-148"><a href="#cb27-148" aria-hidden="true" tabindex="-1"></a>            imgs <span class="op">=</span> imgs.to(DEVICE)</span>
<span id="cb27-149"><a href="#cb27-149" aria-hidden="true" tabindex="-1"></a>            embs <span class="op">=</span> model(imgs)</span>
<span id="cb27-150"><a href="#cb27-150" aria-hidden="true" tabindex="-1"></a>            emb_np, lbl_np <span class="op">=</span> embs.detach().cpu().numpy(), lbls.numpy()</span>
<span id="cb27-151"><a href="#cb27-151" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-152"><a href="#cb27-152" aria-hidden="true" tabindex="-1"></a>            <span class="co"># Hard negative mining via Milvus</span></span>
<span id="cb27-153"><a href="#cb27-153" aria-hidden="true" tabindex="-1"></a>            neg_np <span class="op">=</span> mine_negatives(client, COLLECTION_NAME, emb_np, lbl_np)</span>
<span id="cb27-154"><a href="#cb27-154" aria-hidden="true" tabindex="-1"></a>            neg <span class="op">=</span> F.normalize(torch.tensor(neg_np, device<span class="op">=</span>DEVICE), p<span class="op">=</span><span class="dv">2</span>, dim<span class="op">=</span><span class="dv">1</span>)</span>
<span id="cb27-155"><a href="#cb27-155" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-156"><a href="#cb27-156" aria-hidden="true" tabindex="-1"></a>            <span class="co"># Build positives from same-class shuffling</span></span>
<span id="cb27-157"><a href="#cb27-157" aria-hidden="true" tabindex="-1"></a>            pos_idx <span class="op">=</span> torch.arange(<span class="bu">len</span>(lbls))</span>
<span id="cb27-158"><a href="#cb27-158" aria-hidden="true" tabindex="-1"></a>            <span class="cf">for</span> lbl <span class="kw">in</span> lbls.unique():</span>
<span id="cb27-159"><a href="#cb27-159" aria-hidden="true" tabindex="-1"></a>                m <span class="op">=</span> (lbls <span class="op">==</span> lbl).nonzero(as_tuple<span class="op">=</span><span class="va">True</span>)[<span class="dv">0</span>]</span>
<span id="cb27-160"><a href="#cb27-160" aria-hidden="true" tabindex="-1"></a>                <span class="cf">if</span> <span class="bu">len</span>(m) <span class="op">&gt;</span> <span class="dv">1</span>:</span>
<span id="cb27-161"><a href="#cb27-161" aria-hidden="true" tabindex="-1"></a>                    pos_idx[m] <span class="op">=</span> m[torch.randperm(<span class="bu">len</span>(m))]</span>
<span id="cb27-162"><a href="#cb27-162" aria-hidden="true" tabindex="-1"></a>            pos <span class="op">=</span> embs[pos_idx]</span>
<span id="cb27-163"><a href="#cb27-163" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-164"><a href="#cb27-164" aria-hidden="true" tabindex="-1"></a>            loss <span class="op">=</span> loss_fn(embs, pos, neg)</span>
<span id="cb27-165"><a href="#cb27-165" aria-hidden="true" tabindex="-1"></a>            opt.zero_grad()<span class="op">;</span> loss.backward()</span>
<span id="cb27-166"><a href="#cb27-166" aria-hidden="true" tabindex="-1"></a>            torch.nn.utils.clip_grad_norm_(model.parameters(), <span class="fl">1.0</span>)</span>
<span id="cb27-167"><a href="#cb27-167" aria-hidden="true" tabindex="-1"></a>            opt.step()</span>
<span id="cb27-168"><a href="#cb27-168" aria-hidden="true" tabindex="-1"></a>            total_loss <span class="op">+=</span> loss.item()</span>
<span id="cb27-169"><a href="#cb27-169" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-170"><a href="#cb27-170" aria-hidden="true" tabindex="-1"></a>        sched.step()</span>
<span id="cb27-171"><a href="#cb27-171" aria-hidden="true" tabindex="-1"></a>        <span class="bu">print</span>(<span class="ss">f"Epoch </span><span class="sc">{</span>epoch<span class="sc">:3d}</span><span class="ss"> | Loss=</span><span class="sc">{</span>total_loss<span class="op">/</span><span class="bu">len</span>(train_dl)<span class="sc">:.4f}</span><span class="ss">"</span>)</span>
<span id="cb27-172"><a href="#cb27-172" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-173"><a href="#cb27-173" aria-hidden="true" tabindex="-1"></a>        <span class="cf">if</span> epoch <span class="op">%</span> REINDEX_EVERY <span class="op">==</span> <span class="dv">0</span>:</span>
<span id="cb27-174"><a href="#cb27-174" aria-hidden="true" tabindex="-1"></a>            <span class="bu">print</span>(<span class="st">"Re-indexing training set..."</span>)</span>
<span id="cb27-175"><a href="#cb27-175" aria-hidden="true" tabindex="-1"></a>            client.delete(COLLECTION_NAME, <span class="bu">filter</span><span class="op">=</span><span class="st">'split == "train"'</span>)</span>
<span id="cb27-176"><a href="#cb27-176" aria-hidden="true" tabindex="-1"></a>            client.flush(COLLECTION_NAME)</span>
<span id="cb27-177"><a href="#cb27-177" aria-hidden="true" tabindex="-1"></a>            embed_and_store(model, train_dl, <span class="st">"train"</span>, client, COLLECTION_NAME, DEVICE)</span>
<span id="cb27-178"><a href="#cb27-178" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-179"><a href="#cb27-179" aria-hidden="true" tabindex="-1"></a>        torch.save({<span class="st">"epoch"</span>: epoch, <span class="st">"model"</span>: model.state_dict()},</span>
<span id="cb27-180"><a href="#cb27-180" aria-hidden="true" tabindex="-1"></a>                   <span class="ss">f"</span><span class="sc">{</span>CHECKPOINT_DIR<span class="sc">}</span><span class="ss">/encoder_</span><span class="sc">{</span>epoch<span class="sc">:03d}</span><span class="ss">.pt"</span>)</span>
<span id="cb27-181"><a href="#cb27-181" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-182"><a href="#cb27-182" aria-hidden="true" tabindex="-1"></a>    <span class="bu">print</span>(<span class="st">"Training complete."</span>)</span>
<span id="cb27-183"><a href="#cb27-183" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-184"><a href="#cb27-184" aria-hidden="true" tabindex="-1"></a></span>
<span id="cb27-185"><a href="#cb27-185" aria-hidden="true" tabindex="-1"></a><span class="cf">if</span> <span class="va">__name__</span> <span class="op">==</span> <span class="st">"__main__"</span>:</span>
<span id="cb27-186"><a href="#cb27-186" aria-hidden="true" tabindex="-1"></a>    main()</span></code></pre></div></div>

<section id="summary" class="level2">
<h2 class="anchored" data-anchor-id="summary" id="summary">Summary</h2>
<p>This guide walked through the complete lifecycle of combining PyTorch and Milvus:</p>
<ul>
<li><strong>Milvus</strong> acts as an external memory that stores your model’s evolving embedding space and enables fast approximate nearest neighbor retrieval during training.</li>
<li><strong>PyTorch</strong> handles the gradient-based optimization and forward/backward passes.</li>
<li><strong>Hard negative mining</strong> via Milvus dramatically accelerates metric learning convergence compared to random negatives.</li>
<li><strong>Periodic re-indexing</strong> keeps the stored embeddings consistent with the changing model weights.</li>
<li>The same Milvus collection used during training transitions seamlessly into a production search index after training completes.</li>
</ul>
