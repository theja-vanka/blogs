<section id="executive-summary" class="level2">
<h2 class="anchored" data-anchor-id="executive-summary" id="executive-summary">Executive summary</h2>
<p><strong>Segment Anything (SA)</strong> is a project — not merely a model — that ports the “foundation model” recipe from NLP to image segmentation. It is built on three tightly coupled deliverables:</p>
<ul>
<li><strong>A task</strong> — <em>promptable segmentation</em>: return a <em>valid</em> mask for <em>any</em> prompt (point, box, mask, or text), even when the prompt is ambiguous.</li>
<li><strong>A model</strong> — <strong>SAM</strong> (Segment Anything Model): a heavyweight image encoder, a lightweight prompt encoder, and a fast mask decoder, designed so the expensive image embedding is computed once and reused across many cheap (~50 ms) prompt queries in a browser.</li>
<li><strong>A dataset</strong> — <strong>SA-1B</strong>: over <strong>1.1 billion masks</strong> on <strong>11 million</strong> licensed, privacy-protecting images — roughly <strong>400× more masks</strong> than any prior segmentation dataset — collected by a <strong>data engine</strong> that loops model and annotators.</li>
</ul>
<p>The headline result is that SAM transfers <strong>zero-shot</strong> to a wide range of segmentation tasks and image distributions via <em>prompt engineering</em>, frequently matching or exceeding prior fully supervised methods without any task-specific training. SAM and SA-1B were released openly (Apache 2.0 for the model) to catalyze research into vision foundation models.</p>

<section id="motivation-foundation-models-for-vision" class="level2">
<h2 class="anchored" data-anchor-id="motivation-foundation-models-for-vision" id="motivation-foundation-models-for-vision">Motivation: foundation models for vision</h2>
<p>Large language models pre-trained on web-scale corpora exhibit strong <strong>zero-shot</strong> and <strong>few-shot</strong> generalization, typically accessed through <strong>prompt engineering</strong>. Their zero/few-shot behavior scales predictably with model size, dataset size, and compute, and can rival fine-tuned models.</p>
<p>Vision has seen analogous progress, most prominently in <strong>image–text alignment</strong> models such as CLIP and ALIGN, which use contrastive learning to produce encoders that generalize zero-shot to new visual concepts and compose with other modules (e.g., DALL·E). But much of computer vision lies beyond image–text alignment, and for many of those problems <strong>abundant training data simply does not exist</strong>. Segmentation is a prime example: there is no web-scale source of masks.</p>
<p>The authors set an explicit goal: <strong>build a foundation model for image segmentation</strong> — a <em>promptable</em> model, pre-trained on a broad dataset via a task that enables powerful generalization, so that downstream segmentation problems on new data distributions can be solved through prompt engineering. Success hinges on jointly answering three entangled questions:</p>
<ol type="1">
<li><strong>What task</strong> enables zero-shot generalization?</li>
<li><strong>What model architecture</strong> supports it?</li>
<li><strong>What data</strong> can power that task and model?</li>
</ol>
</section>
<section id="the-three-interconnected-components" class="level2">
<h2 class="anchored" data-anchor-id="the-three-interconnected-components" id="the-three-interconnected-components">The three interconnected components</h2>
<p>The design is deliberately circular: the <strong>task</strong> defines a pre-training objective and a prompting interface; the <strong>model</strong> must be flexible and fast enough to serve both interactive annotation and downstream prompting; and the <strong>data engine</strong> uses the model to collect data that in turn improves the model.</p>

<section id="the-promptable-segmentation-task" class="level2">
<h2 class="anchored" data-anchor-id="the-promptable-segmentation-task" id="the-promptable-segmentation-task">The promptable segmentation task</h2>
<section id="definition" class="level3">
<h3 class="anchored" data-anchor-id="definition" id="definition">Definition</h3>
<p>Borrowing from NLP’s next-token-prediction paradigm — one objective that serves <em>both</em> pre-training <em>and</em> downstream transfer — the authors define <strong>promptable segmentation</strong>:</p>
<blockquote class="blockquote">
<p>Given any <strong>prompt</strong>, return a <strong>valid segmentation mask</strong>.</p>
</blockquote>
<p>A <em>prompt</em> specifies <strong>what to segment</strong>: a set of foreground/background points, a rough box, a coarse mask, free-form text, or any combination. <strong>Validity</strong> is the crucial subtlety: when a prompt is <strong>ambiguous</strong> (a single point on a shirt could indicate the shirt <em>or</em> the person), the output need only be a <em>reasonable mask for at least one</em> plausible object — much as a language model should still return a coherent response to an ambiguous query.</p>
</section>
<section id="pre-training" class="level3">
<h3 class="anchored" data-anchor-id="pre-training" id="pre-training">Pre-training</h3>
<p>The task implies a natural pre-training algorithm adapted from <strong>interactive segmentation</strong>: for each training mask, simulate a <em>sequence</em> of prompts and compare the model’s predictions to ground truth across the sequence. The key departure from classic interactive segmentation is the objective — rather than aiming to eventually produce a good mask after <em>enough</em> user input, SAM must always produce a valid mask for <em>any</em> prompt, <strong>including ambiguous ones</strong>. This property is what later makes fully automatic annotation feasible.</p>
</section>
<section id="zero-shot-transfer-via-prompting" class="level3">
<h3 class="anchored" data-anchor-id="zero-shot-transfer-via-prompting" id="zero-shot-transfer-via-prompting">Zero-shot transfer via prompting</h3>
<p>Because the model can respond appropriately to any prompt, downstream tasks reduce to <strong>engineering the right prompt</strong>. For example, given a cat bounding-box detector, cat <em>instance segmentation</em> is solved by feeding each detected box to SAM as a prompt. This is <strong>task generalization</strong> — importantly different from <strong>multi-task</strong> systems, which perform a <em>fixed</em> set of tasks whose training and test tasks coincide. SAM instead acts as a <strong>composable component</strong> inside larger systems, potentially serving tasks unknown at design time.</p>

<section id="the-segment-anything-model-sam" class="level2">
<h2 class="anchored" data-anchor-id="the-segment-anything-model-sam" id="the-segment-anything-model-sam">The Segment Anything Model (SAM)</h2>
<section id="design-constraints" class="level3">
<h3 class="anchored" data-anchor-id="design-constraints" id="design-constraints">Design constraints</h3>
<p>Three requirements flow from the task and from the goal of real-world interactive use:</p>
<ul>
<li><strong>Flexible prompts</strong> — points, boxes, masks, and text must all be accepted.</li>
<li><strong>Amortized real-time</strong> — masks must be computed fast enough for interactive prompting.</li>
<li><strong>Ambiguity-aware</strong> — a single ambiguous prompt should yield multiple valid masks.</li>
</ul>
<p>A surprisingly simple three-part design satisfies all three.</p>
<div id="fig-arch-original" class="quarto-float quarto-figure quarto-figure-center anchored">
<figure class="quarto-float quarto-float-fig figure">
<div aria-describedby="fig-arch-original-caption-0ceaefa1-69ba-4598-a22c-09a6ac19f8ca">

Figure&nbsp;4: Figure 4: A heavyweight image encoder produces an image embedding that is then efficiently queried by various prompts to produce masks at amortized real-time speed. Ambiguous prompts yield multiple masks, each with a confidence (IoU) score.
</figcaption>
</figure>
</div>
</section>
<section id="architecture" class="level3">
<h3 class="anchored" data-anchor-id="architecture" id="architecture">Architecture</h3>
<div class="cell" data-eval="true" data-layout-align="center">
<div class="cell-output-display">
<div id="fig-arch" class="quarto-float quarto-figure quarto-figure-center anchored">
<figure class="quarto-float quarto-float-fig figure">
<div aria-describedby="fig-arch-caption-0ceaefa1-69ba-4598-a22c-09a6ac19f8ca">
<div>
<pre class="mermaid mermaid-js" data-label="fig-arch">flowchart LR
    IMG["Input image"] --&gt; IE["Image encoder&lt;br/&gt;(MAE-pretrained ViT-H,&lt;br/&gt;adapted for high-res)"]
    IE --&gt; EMB["Image embedding&lt;br/&gt;(computed once per image)"]

    subgraph PE["Prompt encoder"]
        SPARSE["Sparse: points, box, text&lt;br/&gt;positional enc. + learned emb.&lt;br/&gt;(text via CLIP encoder)"]
        DENSE["Dense: mask&lt;br/&gt;convolution, summed with image emb."]
    end

    EMB --&gt; DEC
    PE --&gt; DEC["Mask decoder&lt;br/&gt;(2x modified Transformer blocks:&lt;br/&gt;prompt self-attn + bidirectional cross-attn)"]
    DEC --&gt; UP["Upsample + dynamic linear classifier (MLP)"]
    UP --&gt; MASKS["3 output masks&lt;br/&gt;+ IoU confidence per mask"]
</pre>
</div>
</div>
<figcaption class="quarto-float-caption-bottom quarto-float-caption quarto-float-fig" id="fig-arch-caption-0ceaefa1-69ba-4598-a22c-09a6ac19f8ca">
Figure&nbsp;5: SAM architecture: heavyweight image encoder + lightweight prompt encoder + fast mask decoder.
</figcaption>
</figure>
</div>
</div>
</div>
<p><strong>Image encoder.</strong> An <strong>MAE-pretrained Vision Transformer (ViT)</strong>, minimally adapted to process high-resolution inputs. It runs <strong>once per image</strong>; its output embedding is cached and reused for every subsequent prompt — the crux of the “amortized real-time” property.</p>
<p><strong>Prompt encoder.</strong> Two families:</p>
<ul>
<li><strong>Sparse</strong> prompts (points, boxes, text): points and boxes are represented by <strong>positional encodings summed with learned type-specific embeddings</strong>; free-form text uses an <strong>off-the-shelf CLIP text encoder</strong>.</li>
<li><strong>Dense</strong> prompts (masks): embedded via <strong>convolutions</strong> and <strong>summed element-wise with the image embedding</strong>.</li>
</ul>
<p><strong>Mask decoder.</strong> A lightweight, modified <strong>Transformer decoder</strong> that maps the image embedding, prompt embeddings, and a learned <strong>output token</strong> to a mask. Each of its <strong>two blocks</strong> applies prompt self-attention plus <strong>cross-attention in both directions</strong> (prompt→image and image→prompt), updating <em>all</em> embeddings. The image embedding is then upsampled, and an MLP turns the output token into a <strong>dynamic linear classifier</strong> that computes a foreground probability at every location.</p>
</section>
<section id="resolving-ambiguity" class="level3">
<h3 class="anchored" data-anchor-id="resolving-ambiguity" id="resolving-ambiguity">Resolving ambiguity</h3>
<p>With a single output, the model would <strong>average</strong> conflicting valid masks under ambiguity. Instead SAM predicts <strong>three masks per prompt</strong> — empirically sufficient, since nested masks are typically at most three deep (<strong>whole → part → subpart</strong>). During training, loss is backpropagated only through the <strong>minimum-loss</strong> mask (a “hindsight”/multiple-choice loss). Each mask also gets a predicted <strong>confidence score (estimated IoU)</strong> used to rank outputs.</p>

<section id="efficiency" class="level3">
<h3 class="anchored" data-anchor-id="efficiency" id="efficiency">Efficiency</h3>
<p>Given a precomputed image embedding, the <strong>prompt encoder + mask decoder run in ~50 ms in a web browser on CPU</strong>, enabling seamless interactive prompting. The heavy ViT image encoder is the expensive part, but its cost is amortized across all prompts on that image.</p>
</section>
<section id="losses-and-training" class="level3">
<h3 class="anchored" data-anchor-id="losses-and-training" id="losses-and-training">Losses and training</h3>
<p>Mask supervision combines <strong>focal loss</strong> and <strong>dice loss</strong>. Training simulates an interactive setup by <strong>randomly sampling prompts over 11 rounds per mask</strong>, so SAM integrates directly into the data engine’s annotation loop. Text prompts are handled separately (see <a href="#sec-text" class="quarto-xref">Section&nbsp;1.9.5</a>).</p>
</section>
</section>

<section id="stage-1-assisted-manual" class="level3">
<h3 class="anchored" data-anchor-id="stage-1-assisted-manual" id="stage-1-assisted-manual">Stage 1 — Assisted-manual</h3>
<p>Professional annotators label masks by clicking foreground/background points in a <strong>browser tool powered by SAM</strong> (using precomputed embeddings for real-time interaction), refining with pixel-precise brush/eraser tools. <strong>No semantic constraints</strong> — annotators freely label both “stuff” and “things,” labeling in order of prominence and moving on after ~30 s per mask. SAM was initially trained on public datasets, then <strong>retrained only on newly collected masks</strong>; over the stage the encoder scaled <strong>ViT-B → ViT-H</strong> and the model was retrained <strong>6 times</strong>. Average time per mask fell from <strong>34 s to 14 s</strong> (≈6.5× faster than COCO polygon annotation), and masks/image rose from <strong>~20 to ~44</strong>. Yield: <strong>4.3M masks from 120k images</strong>.</p>
</section>
<section id="stage-2-semi-automatic" class="level3">
<h3 class="anchored" data-anchor-id="stage-2-semi-automatic" id="stage-2-semi-automatic">Stage 2 — Semi-automatic</h3>
<p>Goal: increase <strong>mask diversity</strong>. A generic “object” <strong>bounding-box detector</strong> trained on stage-1 masks pre-fills confident masks; annotators then focus on the <strong>remaining, less prominent objects</strong>. The model was retrained <strong>5 times</strong>; time/mask returned to ~34 s (harder objects), and masks/image rose <strong>44 → 72</strong> (including automatic masks). Added <strong>5.9M masks in 180k images</strong> (10.2M total).</p>
</section>
<section id="stage-3-fully-automatic" class="level3">
<h3 class="anchored" data-anchor-id="stage-3-fully-automatic" id="stage-3-fully-automatic">Stage 3 — Fully automatic</h3>
<p>Now feasible because (a) enough diverse masks had been collected and (b) the model was <strong>ambiguity-aware</strong>. SAM is prompted with a <strong>32×32 regular grid of points</strong>; each point yields masks for whole/part/subpart. Masks are filtered by <strong>predicted IoU (confidence)</strong> and <strong>stability</strong> (thresholding the probability map at 0.5±δ yields similar masks), then de-duplicated with <strong>NMS</strong>; <strong>zoomed-in crops</strong> improve small masks. Applied to all <strong>11M images</strong>, producing <strong>1.1B masks</strong>.</p>
</section>
</section>
<section id="the-sa-1b-dataset" class="level2">
<h2 class="anchored" data-anchor-id="the-sa-1b-dataset" id="the-sa-1b-dataset">The SA-1B dataset</h2>
<p><strong>SA-1B</strong> = 11M licensed, privacy-protecting images + 1.1B high-quality masks.</p>
<ul>
<li><strong>Images.</strong> Licensed from a provider working directly with photographers; average native resolution <strong>~3300×4950 px</strong>. Released <strong>downsampled to 1500 px shortest side</strong> (still far larger than, e.g., COCO’s ~480×640). <strong>Faces and license plates are blurred.</strong></li>
<li><strong>Masks.</strong> <strong>99.1% generated fully automatically.</strong> SA-1B ships <strong>only automatic masks</strong>, justified by quality analysis.</li>
<li><strong>Mask quality.</strong> On ~500 images (~50k masks), professionals corrected the automatic masks. <strong>94% of pairs exceed 90% IoU; 97% exceed 75% IoU</strong> — comparable to or better than reported <strong>inter-annotator consistency (85–91% IoU)</strong>. Training on automatic masks alone is nearly as good as using all data-engine masks.</li>
<li><strong>Scale.</strong> <strong>11× more images and 400× more masks</strong> than Open Images (the next largest); <strong>~36×</strong> more masks per image; even ADE20K has <strong>3.5×</strong> fewer masks/image.</li>
<li><strong>Mask properties.</strong> Better coverage of <strong>image corners</strong> than LVIS/ADE20K (COCO and Open Images show stronger center bias); more small/medium relative-size masks; <strong>concavity (shape-complexity) distribution similar</strong> to other datasets after controlling for size.</li>
</ul>

<section id="single-point-valid-mask-segmentation" class="level3">
<h3 class="anchored" data-anchor-id="single-point-valid-mask-segmentation" id="single-point-valid-mask-segmentation">Single-point valid-mask segmentation</h3>
<p>The hardest, most ambiguous prompt: <strong>one foreground point</strong>. Because ground truth rarely enumerates all valid masks, automatic <strong>mIoU is supplemented by a human study</strong> (annotators rate masks 1–10). Points are sampled from the mask “center” (distance-transform maximum); SAM reports its <strong>most confident</strong> mask by default. The main baseline is <strong>RITM</strong>, a strong interactive segmenter.</p>
<p><strong>Results.</strong> SAM beats RITM on <strong>16 of 23 datasets</strong> by automatic mIoU (by up to ~47 IoU). With an <strong>“oracle”</strong> that picks the best of SAM’s 3 masks, SAM beats RITM on <strong>all 23</strong>. In the <strong>human study</strong>, annotators consistently rate SAM’s masks <strong>substantially higher</strong> than RITM (mean ratings ~7–9); an <strong>ambiguity-unaware, single-mask ablation</strong> scores lower than full SAM but still above RITM. Notably, on datasets where SAM <em>loses</em> on automatic metrics (e.g., DRAM, IBD), humans still rate it higher — illustrating the unreliability of automatic single-point metrics under ambiguity. As prompts grow from 1→9 points, methods converge (SAM is not tuned for the very-high-IoU regime); with <strong>random</strong> (rather than center) point sampling, SAM’s advantage <strong>grows</strong>.</p>

<section id="zero-shot-edge-detection" class="level3">
<h3 class="anchored" data-anchor-id="zero-shot-edge-detection" id="zero-shot-edge-detection">Zero-shot edge detection</h3>
<p><strong>Approach.</strong> A simplified auto-mask pipeline: prompt SAM with a <strong>16×16 grid</strong> (768 masks, 3/point), NMS, then compute edges via <strong>Sobel filtering of unthresholded mask probability maps</strong> + lightweight edge NMS. SAM never saw BSDS500 images or edge labels.</p>
<p><strong>Results (BSDS500).</strong> SAM produces reasonable edge maps and <strong>outperforms classic zero-shot methods</strong> (Sobel, Canny, Felzenszwalb–Huttenlocher). It trails specialized learned detectors (HED, EDETR) but is competitive given it was never trained for edges. SAM tends to predict <strong>more edges than the ground truth annotates</strong> (it segments things not labeled in BSDS).</p>

<section id="zero-shot-object-proposals" class="level3">
<h3 class="anchored" data-anchor-id="zero-shot-object-proposals" id="zero-shot-object-proposals">Zero-shot object proposals</h3>
<p><strong>Approach.</strong> Run the full automatic mask generator; treat masks as <strong>object proposals</strong>. <strong>Metric:</strong> average recall (AR) on <strong>LVIS v1</strong> (chosen for its large vocabulary). <strong>Baseline:</strong> a strong <strong>ViTDet-H</strong> detector’s proposal stage.</p>
<p><strong>Results.</strong> SAM is <strong>competitive</strong> with the strong supervised baseline. It <strong>outperforms on medium/large objects and on rare/common categories</strong>, but is <strong>worse on small and frequent objects</strong>, where ViTDet — trained specifically on LVIS — can exploit dataset-specific biases SAM never learned.</p>
</section>
<section id="zero-shot-instance-segmentation" class="level3">
<h3 class="anchored" data-anchor-id="zero-shot-instance-segmentation" id="zero-shot-instance-segmentation">Zero-shot instance segmentation</h3>
<p><strong>Approach.</strong> A textbook composition: feed an <strong>object detector’s boxes</strong> (ViTDet) to SAM as prompts to produce instance masks.</p>
<p><strong>Results (COCO, LVIS).</strong> SAM’s mask <strong>AP is slightly below</strong> the fully supervised ViTDet on COCO. But because COCO ground-truth masks are relatively coarse, a <strong>human study rates SAM’s masks higher</strong> than ViTDet’s — SAM’s boundaries are often visibly better than the “ground truth.” On LVIS, whose masks are higher quality, the gap narrows.</p>

<section id="sec-text" class="level3">
<h3 class="anchored" data-anchor-id="sec-text" id="sec-text">Zero-shot text-to-mask (proof of concept)</h3>
<p><strong>Approach.</strong> A preliminary probe of free-form-text prompting. During training, each mask is paired with its <strong>CLIP image embedding</strong> as a prompt; because CLIP aligns image and text, at inference the model can be prompted with <strong>CLIP text embeddings</strong> instead — no explicit text supervision on masks required.</p>
<p><strong>Results.</strong> SAM can segment from prompts like <em>“a wheel”</em> or <em>“beaver tooth grille.”</em> When text alone is imprecise, adding <strong>a point</strong> refines the result. The authors frame this as a <strong>proof of concept</strong>, not a robust text-to-mask system.</p>
</section>
</section>
<section id="ablations" class="level2">
<h2 class="anchored" data-anchor-id="ablations" id="ablations">Ablations</h2>
<p>Studied on the 23-dataset suite (mIoU at one center point):</p>
<ul>
<li><strong>Data engine stages.</strong> Cumulative data helps: training on stages 1→1+2→1+2+3 improves results. Using <strong>only automatic masks</strong> (stage 3) performs comparably to using all stages — the practical justification for SA-1B shipping automatic masks only.</li>
<li><strong>Data scaling.</strong> Scaling images from <strong>~0.1M → 1M → full 11M</strong> improves results with <strong>diminishing returns</strong>; a large fraction of the benefit is realized well before the full dataset.</li>
<li><strong>Image-encoder scaling.</strong> <strong>ViT-B → ViT-L</strong> gives a large jump; <strong>ViT-L → ViT-H</strong> is smaller — the paper’s default ViT-H trades compute for a modest gain.</li>
</ul>
</section>
<section id="discussion-limitations-and-impact" class="level2">
<h2 class="anchored" data-anchor-id="discussion-limitations-and-impact" id="discussion-limitations-and-impact">Discussion, limitations, and impact</h2>
<section id="foundation-model-framing" class="level3">
<h3 class="anchored" data-anchor-id="foundation-model-framing" id="foundation-model-framing">Foundation-model framing</h3>
<p>SAM is positioned as a <strong>promptable, composable</strong> vision foundation model. Its value lies less in any single benchmark number and more in being a <strong>reusable component</strong>: prompt engineering and composition let one model serve many tasks — including ones unanticipated at training time — analogous to how CLIP serves as the alignment module inside DALL·E.</p>
</section>
<section id="limitations" class="level3">
<h3 class="anchored" data-anchor-id="limitations" id="limitations">Limitations</h3>
<ul>
<li><strong>Fine structure &amp; boundaries.</strong> SAM can miss thin structures, occasionally <strong>hallucinates small disconnected components</strong>, and produces <strong>less crisp boundaries</strong> than compute-intensive methods that “zoom in.”</li>
<li><strong>Interactive high-IoU regime.</strong> Dedicated interactive segmenters can outperform SAM when <strong>many points</strong> are provided; SAM is optimized for generality, not the very-high-IoU regime.</li>
<li><strong>Real-time caveat.</strong> Only the <strong>prompt encoder + mask decoder</strong> are real-time (~50 ms). With the <strong>heavy ViT-H image encoder</strong>, overall processing is <strong>not</strong> real-time.</li>
<li><strong>Text-to-mask is preliminary.</strong> Not robust; needs better modeling.</li>
<li><strong>Domain-specific tools</strong> may still beat SAM within their domains.</li>
<li><strong>Semantic/panoptic ambiguity.</strong> It is unclear how to design <strong>simple prompts</strong> for semantic and panoptic segmentation, and “what counts as an object” is inherently ambiguous.</li>
</ul>
</section>
<section id="impact" class="level3">
<h3 class="anchored" data-anchor-id="impact" id="impact">Impact</h3>
<p>By releasing <strong>SAM (Apache 2.0)</strong> and <strong>SA-1B</strong>, the authors aim to make promptable segmentation a <strong>building block</strong> for larger systems and to seed further vision-foundation-model research. SAM catalyzed a large ecosystem of downstream work (medical imaging, video, annotation tooling, and follow-ups such as HQ-SAM, MobileSAM, and SAM 2).</p>
</section>
</section>
<section id="key-takeaways-for-practitioners" class="level2">
<h2 class="anchored" data-anchor-id="key-takeaways-for-practitioners" id="key-takeaways-for-practitioners">Key takeaways for practitioners</h2>
<ul>
<li><strong>Amortize the encoder.</strong> Compute the image embedding <strong>once</strong>, then issue many cheap prompts — the design pattern that makes interactive use practical.</li>
<li><strong>Exploit the 3-mask output.</strong> For ambiguous prompts, use the <strong>IoU scores</strong> to pick or expose whole/part/subpart, rather than forcing a single averaged mask.</li>
<li><strong>Compose, don’t fine-tune.</strong> Many tasks (instance segmentation, proposals, edges) are solvable by <strong>prompt engineering + composition</strong> with existing detectors/encoders — no SAM retraining needed.</li>
<li><strong>Mind the caveats.</strong> Expect softer boundaries and possible stray components on fine structures; add points to disambiguate; the encoder dominates latency.</li>
<li><strong>Automatic masks are enough.</strong> SA-1B’s automatic-only masks train strong models — a template for <strong>model-in-the-loop data engines</strong> in data-scarce domains.</li>
</ul>
</section>
<section id="glossary" class="level2">
<h2 class="anchored" data-anchor-id="glossary" id="glossary">Glossary</h2>
<ul>
<li><strong>Promptable segmentation</strong> — the task of returning a valid mask for any prompt, including ambiguous ones.</li>
<li><strong>Valid mask</strong> — a reasonable mask for at least one plausible object indicated by the prompt.</li>
<li><strong>Sparse vs.&nbsp;dense prompts</strong> — sparse = points/boxes/text (positional + learned embeddings, text via CLIP); dense = masks (convolutional embeddings summed with the image embedding).</li>
<li><strong>Ambiguity-aware</strong> — predicting multiple (3) masks per prompt, ranked by predicted IoU; trained with a minimum-over-masks loss.</li>
<li><strong>Amortized real-time</strong> — expensive image embedding computed once and reused across many fast prompt queries (~50 ms each).</li>
<li><strong>Data engine</strong> — model-in-the-loop annotation across assisted-manual, semi-automatic, and fully-automatic stages.</li>
<li><strong>Stability</strong> — a mask is stable if thresholding its probability map at 0.5±δ yields similar masks.</li>
<li><strong>Zero-shot transfer</strong> — applying SAM to tasks/distributions unseen in training via prompt engineering (CLIP sense).</li>
<li><strong>SA-1B</strong> — the released dataset: 11M images, 1.1B automatic masks.</li>
</ul>
